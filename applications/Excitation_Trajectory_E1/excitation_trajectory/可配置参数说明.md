# 激励轨迹可配置参数说明

以下为 **Standalone**（`run_excitation_trajectory_standalone.m`）与 **TLBO_E1**（`TLBO_E1/run_TLBO_Excitation_E1.m`）中可修改、设置的参数及作用。固定逻辑（如设计 0.4、校核 0.45）未列入。

---

## 一、Standalone 方案（run_excitation_trajectory_standalone.m）

### 1. 运行模式
| 参数 | 类型 | 默认 | 作用 |
|------|------|------|------|
| **FAST_VERIFY** | logical | true | true：减少周期数、优化迭代/评估上限，快速跑通；正式生成建议 false。 |

### 2. 肢体与维度
| 参数 | 类型 | 默认/示例 | 作用 |
|------|------|-----------|------|
| **config.move_axis** | 1×n | [0,1,2,3,4,5] | 参与运动的关节下标（0 起），6=左腿，3=单臂。 |
| **config.init_joint** | n×1 (rad) | zeros(6,1) | 轨迹起点/终点关节角；傅里叶常数项取 (upper+lower)/2 时会被覆盖。 |
| **n_right** | 标量 | 6 | 输出时右腿补零列数；单臂时脚本内会置 0。 |
| **config.cond_limb** | 字符串 | 'left_leg' | 用于条件数与 robot：'left_leg'/'right_leg'/'left_arm'/'right_arm'。 |

### 3. 位置 / 速度 / 加速度限值（名义全范围）
| 参数 | 类型 | 单位 | 作用 |
|------|------|------|------|
| **config.upper_joint_bound** | n×1 | rad | 关节位置上界（名义全范围）；设计=完整范围的 0.4、校核=完整范围的 0.45，均由此算。 |
| **config.lower_joint_bound** | n×1 | rad | 关节位置下界（名义全范围）。 |
| **config.max_velocity** | n×1 | rad/s | 关节速度上限；优化约束与校核均用。 |
| **config.max_acceleration** | n×1 | rad/s² | 关节加速度上限；优化约束与校核均用。 |

### 4. 轨迹形状与采样
| 参数 | 类型 | 默认 | 作用 |
|------|------|------|------|
| **config.traj_cycle** | 标量 | 5 | **周期重复次数**：完整轨迹 = 过渡段 + 单周期重复 traj_cycle 次 + 过渡段；要多周期请设 >1（快速验证时脚本会改为 1）。 |
| **config.sample_frequency** | 标量 | 100 Hz | 采样频率；单周期点数 ≈ period×sample_frequency。 |
| **config.order** | 标量 | 6 | 傅里叶阶数；阶数越大轨迹越丰富，速度/加速度易增大。 |
| **config.period** | 标量 (s) | 8 | 单周期时长 T；ω=2π/T；**减小 period** 可提高 ω，从而提高速度/加速度幅值。 |
| **config.sample_number** | 标量 | 15 | 优化时约束离散点数（每周期）。 |

### 5. 优化与约束
| 参数 | 类型 | 默认 | 作用 |
|------|------|------|------|
| **config.amplitude_alpha** | 标量 | 0.7 | 幅值约束：sum(\|a\|+\|b\|) ≤ alpha×(upper-lower)/2，0.6~0.8 常用。 |
| **config.use_full_bounds_for_optim** | logical | true | true：优化与校核用 config 原始限位（不收紧）；false：设计=完整范围的 0.4、速度=0.9×上限。 |
| **config.coeff_lower / coeff_upper** | 标量 | -0.5, 0.5 | 傅里叶系数 a、b 的优化上下界；放宽可增大速度/加速度，受 amplitude_alpha 限制。 |
| **config.enable_collision_check** | logical | true | true：优化与校验做自碰撞检测（需 Robotics System Toolbox）；false：不做。 |

### 6. 扭矩与机器人（可选覆盖）
| 参数 | 类型 | 说明 |
|------|------|------|
| **config.robot_limb** | rigidBodyTree | 不设则按 cond_limb 用 get_e1_limb_robot 自动生成。 |
| **config.max_effort** | n×1 (N·m) | 不设则从 URDF 或 E1 腿默认 [110;170;170;330;70;70] 读取；校核限值=0.9×max_effort。 |

### 7. 条件数（可选）
| 参数 | 类型 | 说明 |
|------|------|------|
| **config.regressor_source** | 字符串 | 默认 'E1_limb'，与 cond_limb 一起决定条件数函数。 |
| **config.cond_limb** | 字符串 | 见上；与 regressor_source 决定 cond_fun。 |

## 二、TLBO_E1 方案（TLBO_E1/run_TLBO_Excitation_E1.m）

### 1. 肢体与名义限位
| 参数 | 类型 | 默认/示例 | 作用 |
|------|------|-----------|------|
| **limb** | 字符串 | 'left_leg' | 'left_leg'/'right_leg'/'left_arm'/'right_arm'；决定 DOF 与 robot_limb。 |
| **LARGE_EXCITATION** | logical | false | true：位置名义限位 ±2π，缩放由 dq_max/ddq_max 主导，易顶满速度/加速度；false：紧限位（E1 腿典型值）。 |
| **q_max, q_min** | DOF×1 (rad) | 见脚本 | 名义位置上下界（全范围）；设计=中间点±0.4×半范围、校核=±0.45×半范围由此算。 |
| **dq_max** | DOF×1 | rad/s | 速度上限。 |
| **ddq_max** | DOF×1 | rad/s² | 加速度上限。 |

说明：设计用 `q_max_design`/`q_min_design`（中间点±0.4×半范围）由脚本从 q_max/q_min 自动计算，无需手改。

### 2. 周期与采样
| 参数 | 类型 | 默认 | 作用 |
|------|------|------|------|
| **period** | 标量 (s) | 4 | 单周期时长；缩短可提高 ω，从而增大速度/加速度幅值。 |
| **sample_frequency** | 标量 (Hz) | 500 | 采样频率；条件数计算与校核采样点数由此定。 |
| **traj_cycle** | 标量 | 5 | **周期重复次数**：优化与校核仍按单周期；导出 CSV/完整轨迹时把单周期重复 traj_cycle 次；设 >1 即得多周期轨迹。 |
| 设计/校核位置 | 固定 | 0.4 / 0.45 | 设计=**完整范围（q_max−q_min）的 0.4**；校核=**完整范围的 0.45**（均以中间点对称）。 |

### 3. TLBO 算法
| 参数 | 类型 | 默认 | 作用 |
|------|------|------|------|
| **Population** | 整数 | 10 | 种群规模。 |
| **Iteration** | 整数 | 20 | 迭代次数。 |
| **use_constraint_penalty** | logical | true | true：约束违反量乘以 penalty_scale 加到目标值，促使满足位置/速度/加速度/碰撞。 |
| **penalty_scale** | 标量 | 10000 | 约束违反惩罚系数。 |

### 4. 碰撞与扭矩
| 参数 | 类型 | 默认 | 作用 |
|------|------|------|------|
| **Enable_Collision_Check** | logical | true | true：校核与（若 use_constraint_penalty）约束惩罚均做自碰撞检测。 |
| **max_effort** | DOF×1 (N·m) | 从 URDF 或 E1 默认 | 峰值力矩；校核限值=0.9×max_effort；脚本开头可改 URDF 路径或兜底默认。 |
| **TORQUE_FEASIBILITY_RATIO** | 标量 | 0.9 | **扭矩可行化**：每次候选解在 Tradeoff_Modify 之后再用 λ_τ 缩放，使 max\|τ_j\| ≤ 该比例×max_effort；与校核限值一致。 |

**TLBO 可行化流程**（每个候选解）：先按关节做 **Tradeoff_Modify**（位置/速度/加速度缩放到限位内），再整体做 **扭矩缩放 λ_τ**（`apply_torque_scale_E1`：若 max\|τ\| 超限则整行系数乘以 λ_τ≤1），保证校核时扭矩不超限。

### 5. 固定不变（仅说明，勿改）
| 项 | 说明 |
|----|------|
| 设计位置范围 | 半宽 = **0.4×完整范围**（upper = center + 0.4×(q_max−q_min)），以中间点对称。 |
| 校核位置范围 | 半宽 = **0.45×完整范围**（pos_valid = center ± 0.45×(q_max−q_min)），以中间点对称。 |
| 傅里叶每关节系数个数 | 11（Num_Design_Variate_OneDof），由阶数等定。 |

## 三、两方案共用逻辑（不通过上述参数修改）

- **位置设计**：半宽 = **0.4×完整范围**（代码：center + 0.4*(upper−lower)）。
- **位置校核**：半宽 = **0.45×完整范围**（代码：center ± 0.45*(upper−lower)）。
- **扭矩校核**：限值 = **0.9**×max_effort（峰值 90%）。
- **速度/加速度校核**：relax_vel=relax_acc=1.0（无放宽）。

若需改 0.4/0.45/0.9 等，需改 `validate_trajectory_E1.m`、`generate_excitation_trajectory.m` 及 TLBO 内设计限位计算处。

## 四、运行时的两个检查点（已接入脚本）

两方案在打印限值/校核参数后都会自动输出：

1. **检查点 1：初值**  
   每关节 **qd(0)、qdd(0)** 的数值（理论上应接近 0，便于周期首尾衔接）。若明显不为 0，说明傅里叶系数或约束需检查。

2. **检查点 2：四峰值与裕度**  
   每关节打印 **max q, min q, max|qd|, max|qdd|**，以及相对限位的**裕度**（最朴素定义，便于判断谁在卡缩放）：  
   - **up_pos** = q_max − max(q)  
   - **lo_pos** = min(q) − q_min  
   - **vel** = dq_max − max|qd|  
   - **acc** = ddq_max − max|qdd|  
   TLBO 的裕度相对**名义限位** q_max/q_min、dq_max、ddq_max；Standalone 相对 config 中的 upper/lower、max_velocity、max_acceleration。裕度大说明该量未贴边，缩放瓶颈在别的量。

## 五、想提高速度/加速度时除了减小 period 还能怎么做？

| 方法 | 适用 | 说明 |
|------|------|------|
| **减小 period** | 两方案 | ω=2π/T 变大，傅里叶里 dq 幅值∝系数、ddq 幅值∝ω×系数，故加速度明显增大，速度也易增大。 |
| **提高 config.order** | Standalone | 傅里叶阶数越大，谐波越多，同一系数界下更容易拼出更大速度/加速度峰值。 |
| **放宽 config.coeff_lower / coeff_upper** | Standalone | 如改为 ±0.8 或 ±1，系数更大，速度/加速度幅值增大；需同时提高 **config.amplitude_alpha**（如 0.8）否则幅值约束会卡住。 |
| **提高 config.amplitude_alpha** | Standalone | 幅值约束 sum(\|a\|+\|b\|) ≤ alpha×(upper−lower)/2；alpha 从 0.7 提到 0.8，允许更大系数，利于提高速度/加速度。 |
| **提高 dq_max / ddq_max** | TLBO_E1 | 约束和 Tradeoff_Modify 按 dq_max、ddq_max 缩放；**增大这两项**后优化允许的轨迹速度/加速度上限提高，易得到更大速度/加速度。 |
| **LARGE_EXCITATION = true** | TLBO_E1 | 位置用 ±2π 宽限位，缩放由 dq_max/ddq_max 主导，轨迹更容易顶满速度/加速度限。 |
| **设计位置范围放宽** | 两方案 | 若把设计从 0.4×完整范围改为更大（如 0.8 或 1.0），可行域更大，优化更容易找到高速度/加速度轨迹（仍受 dq_max/ddq_max 或 coeff 约束）。 |

### 若要更强激励（让轨迹尽量顶到限位）

1. **放大系数初值 + 自动缩回可行域**  
   论文实验里会把傅里叶系数在**较大范围**内随机初始化，再用 Algorithm 1（本仓库里 TLBO 的 **Tradeoff_Modify**）把轨迹自动缩回位置/速度/加速度可行域。  
   - **TLBO**：可把初始化从 `-3 + 6*rand(...)` 改为更大范围（如 `-5 + 10*rand`），仍用 Tradeoff_Modify 压回 q_max_design/q_min_design、dq_max、ddq_max 内。  
   - **Standalone**：可把 **config.coeff_lower / coeff_upper** 放宽（如 ±1），并配合 **config.amplitude_alpha** 略增（如 0.8），优化会在更大系数空间里搜，更容易顶满限位。

2. **加“力矩/速度/加速度”缩放项**  
   在目标或约束里加入一项，促使轨迹尽量顶到「力矩 / 速度 / 加速度」中的某一类边界（例如最小化“到速度上限的裕度”或最大化“最小关节扭矩利用率”），这样优化会主动往边界靠，而不是停在裕度很大的内部。  
   这需要改 TLBO 的 **Objective_E1_limb** 或 Standalone 的 **eval_objective / 约束**，在现有条件数目标上叠加上述缩放项（具体公式可按“最希望顶满的是速度还是力矩”来定）。

## 六、设计/校核位置范围的具体数值（E1 左腿示例）

**约定**：完整范围 = q_max − q_min；设计半宽 = **0.4×完整范围**（即 center ± 0.4×完整范围），校核半宽 = **0.45×完整范围**。

名义全范围（q_max / q_min）取 TLBO 紧限位时（设计半宽=0.4×完整范围，校核半宽=0.45×完整范围）：

| 关节 j | q_min (rad) | q_max (rad) | 完整范围 | 设计下界 | 设计上界 | 校核下界 | 校核上界 |
|--------|-------------|-------------|----------|----------|----------|----------|----------|
| 1 | -0.61 | 0.61 | 1.22 | -0.488 | 0.488 | -0.549 | 0.549 |
| 2 | -0.261 | 0.436 | 0.697 | -0.191 | 0.366 | -0.226 | 0.401 |
| 3 | -2.09 | 0.785 | 2.875 | -1.802 | 0.497 | -1.947 | 0.642 |
| 4 | 0 | 2.44 | 2.44 | 0.244 | 2.196 | 0.122 | 2.318 |
| 5 | -0.872 | 0.523 | 1.395 | -0.733 | 0.384 | -0.802 | 0.454 |
| 6 | -0.262 | 0.262 | 0.524 | -0.210 | 0.210 | -0.236 | 0.236 |

- **设计**：半宽 = 0.4×完整范围，总宽 = 0.8×完整范围；优化/TLBO 只允许轨迹在此范围内。
- **校核**：半宽 = 0.45×完整范围，总宽 = 0.9×完整范围，比设计略宽。

**为什么生成的关节范围变小了？**  
设计用的是**完整范围的 0.4**（40% 宽），优化和 TLBO 的约束都收在这个范围内，所以生成轨迹只会在这个设计范围内。若希望轨迹用满名义全范围，需把设计改为**完整范围的 1.0**（即用回 q_min/q_max），或把 0.4 调大（如 0.8）。

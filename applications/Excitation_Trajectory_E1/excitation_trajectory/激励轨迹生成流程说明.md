# 激励轨迹生成流程说明（超详细）

本文档按**执行顺序**说明从运行 `run_excitation_trajectory_standalone.m` 到得到 12 关节激励轨迹与图的完整流程。涉及两个主文件：**run_excitation_trajectory_standalone.m**（入口与配置）、**generate_excitation_trajectory.m**（核心生成与优化）。

---

## 一、总览：流程分块

```
run_excitation_trajectory_standalone.m
  │
  ├─ 1. 路径与开关
  ├─ 2. 配置 config（关节数、限位、傅里叶 order/period、采样等）
  ├─ 3. robot_limb / max_effort（URDF）
  ├─ 4. cond_fun = cond_fun_from_regressor(config)
  ├─ 5. 循环 try_i = 1 : max_tries
  │     └─ generate_excitation_trajectory(config, ...)
  │           ├─ 5.1 解析 config，收紧可行域，算 COL/CN
  │           ├─ 5.2 确定系数 coeff（给定 / config / 优化 / 随机）
  │           │     若优化 → optimize_fourier_coeff → fmincon
  │           ├─ 5.3 由 coeff 生成单周期 refPos, refVel, refAcc
  │           ├─ 5.4 约束校验 validate_trajectory
  │           └─ 5.5 拼接完整 trajectory（过渡段 + 周期重复 + 回零）
  │     └─ 若 valid_ok 则 break，否则打印超限并重试
  ├─ 6. 扩展为 12 关节（左腿=激励，右腿=0）
  └─ 7. 绘图、打印系数
```

下面按块逐项说明。

---

## 二、入口脚本：run_excitation_trajectory_standalone.m

### 2.1 路径与快速验证开关

- **clear; clc; close all**：清空工作区与图窗。
- **this_dir / parent_dir / repo_root**：当前目录 = `applications/Excitation_Trajectory_E1/excitation_trajectory`，上级 = `Excitation_Trajectory_E1`，仓库根 = `model_e1`。
- **addpath(this_dir, parent_dir)**：保证本目录与 Excitation_Trajectory_E1 在路径上。
- **addpath(fullfile(repo_root, 'applications', 'Body_GravityPara_Iden'))**：显式将 Body_GravityPara_Iden 加入路径，以便调用其中的 `ensure_body_gravity_para_iden_path`、`ReMatrix_E1_limb_URDF` 等。
- **ensure_body_gravity_para_iden_path()**：把 `utility_function`、`dynamics`、`robot_model` 等加进路径，供 `get_e1_limb_robot`、`ReMatrix_E1_limb_URDF`、`inverseDynamics` 等使用。
- **FAST_VERIFY = true/false**：  
  - `true`：快速验证模式。会改写 `config.sample_frequency=100`、`config.traj_cycle=1`，且调用生成时传入 `max_iter_optim=30`、`max_feval_optim=800`，并设 `max_tries=2`。  
  - `false`：正式生成，不改写上述项，`max_tries=5`，优化用默认 150 迭代/5000 评估。

### 2.2 配置 config（含义与用途）

| 字段 | 含义 | 在流程中的用途 |
|------|------|----------------|
| **type** | 辨识类型，10 = BODY_JNT_CURRENT | 仅兼容，生成逻辑未用 |
| **move_axis** | 参与运动的关节下标，如 [0,1,2,3,4,5] | 决定 dim=6（左腿 6 关节） |
| **init_joint** | 各关节初始/终点位置 (rad)，dim×1 | 轨迹起点=终点；傅里叶常数项 q_0 取“可行域中点”后与其一致 |
| **upper_joint_bound / lower_joint_bound** | 关节位置上下限 (rad) | 可行域；校验位置；幅值约束 A_max 用 (upper-lower)/2 |
| **max_velocity / max_acceleration** | 速度、加速度上限 | 约束与校验；优化时在离散点上强制 |qd|≤max_vel、|qdd|≤max_acc |
| **traj_cycle** | 周期重复次数 | 最终 trajectory 里周期段重复几次（如 5 表示重复 5 个周期） |
| **sample_frequency** | 采样频率 (Hz) | 单周期点数 CN = period × sample_frequency；校验与输出都用该密度 |
| **order** | 傅里叶阶数，如 5 | 每关节 sin/cos 项数 k=1..order；系数个数 COL = dim×(2×order+1) |
| **period** | 单周期时长 T (s)，如 10 | 基频 ω=2π/T；单周期时间向量 [0, T] |
| **sample_number** | 约束检查用每周期点数（部分逻辑用） | 与 sample_frequency 同量级，优化里约束点密度另用 25 Hz |
| **amplitude_alpha** | 幅值安全余量，如 0.7 | 每关节 sum(|a_jk|+|b_jk|) ≤ alpha×(upper-lower)/2，保证 |q_j(t)−q_0j| 不超范围 |
| **coeff_lower / coeff_upper** | 系数优化边界，如 [-0.5, 0.5] | fmincon 的 lb/ub（常数项被固定为 init_joint） |
| **enable_collision_check** | 是否做自碰撞检测 | 为 true 时优化约束与校验中会调 checkCollision |
| **robot_limb / max_effort** | 单腿刚体树、关节力矩上限 | 校验时 inverseDynamics 算 τ，要求 |τ|≤max_effort |

### 2.3 robot_limb 与 max_effort

- 若 config 里还没有 **robot_limb** 或 **max_effort**：
  - 用 **get_e1_limb_robot('left_leg')** 得到左腿 rigidBodyTree → **config.robot_limb**。
  - 从仓库中找 **E1.urdf**（如 noetix_description/urdf 或 urdf/），用 **get_limb_effort_from_urdf(urdf_path, 'left_leg')** 得到 6×1 力矩上限 → **config.max_effort**；若某关节读不到则用 50 兜底。
- 这两个在 **validate_trajectory** 里用于扭矩校核：对每个采样点算 inverseDynamics，任一 |τ_j| > max_effort(j) 则校验不通过。

### 2.4 条件数函数 cond_fun

- **config.regressor_source** 默认 `'E1_limb'`，**config.cond_limb** 默认 `'left_leg'`。
- **cond_fun = cond_fun_from_regressor(config)** 得到函数句柄：  
  **cond_fun(refPos, refVel, refAcc)**  
  内部会：用 ReMatrix_E1_limb_URDF 根据 (refPos, refVel, refAcc) 得到观测矩阵 W；列归一化 → rref 取独立列得 W_min；返回 **cond(W_min)**（不返回 cond(W_min'*W_min)，避免平方放大）。
- 若未配置 regressor（如没有 Robot），cond_fun 为空，优化将用“激励量代理”目标（见下）。

### 2.5 循环尝试与调用 generate_excitation_trajectory

- **max_tries**：快速验证为 2，否则为 5。
- 每次尝试：
  - **gen_opts** 固定包含：`'validate', true`，`'optimize', use_optim`（true），`'display_optim', use_optim`，`'cond_fun', cond_fun`。  
    若 FAST_VERIFY，再追加 `'max_iter_optim', 30`，`'max_feval_optim', 800`。
  - 调用  
    **[trajectory, trajPara, refPos, refVel, refAcc, t_period, ~, ~, ~, valid_ok, violation_details] = generate_excitation_trajectory(config, gen_opts{:})**
  - 若 **valid_ok** 为 true，跳出循环；否则打印 violation_details（位置/速度/加速度/扭矩/碰撞等超限），继续下一次尝试。
- **trajectory**：完整时间-关节位置序列 [t, q1..q6]（仅左腿 6 维）；**trajPara**：傅里叶系数向量；**refPos/refVel/refAcc**：单周期 N×6；**t_period**：单周期时间列向量。

### 2.6 扩展为 12 关节与绘图

- **trajectory** 扩展为 [t, q1..q6左, q7..q12右]，右腿 6 关节恒为 0。
- **refPos_full / refVel_full / refAcc_full**：单周期 12 关节（右腿列补 0）。
- 图 1：完整 trajectory 的 12 关节位置随时间。
- 图 2：单周期 12 关节的 refPos / refVel / refAcc。
- 可选保存 MAT；打印系数前 12 个。

以上是入口脚本的完整流程。

## 三、核心函数：generate_excitation_trajectory

### 3.1 输入与可选参数

- **config**：上面所述的结构体。
- 常用可选名值对：
  - **optimize**：是否做系数优化（默认 false）。
  - **coeff**：若给定则直接用该系数生成轨迹，不做优化。
  - **cond_fun**：条件数函数句柄；无则优化目标为“激励量代理” -[ sum(var(refPos))+sum(var(refVel))+sum(var(refAcc)) ]。
  - **max_retries**：优化时随机初值重试次数（默认 5）。
  - **validate**：生成后是否做约束校验（默认 true）。
  - **display_optim**：是否打印 fmincon 迭代（默认 false）。
  - **max_iter_optim / max_feval_optim**：覆盖优化器最大迭代次数与最大函数评估次数（用于快速验证）。

### 3.2 解析 config 与收紧可行域

- **dim** = numel(config.move_axis)（如 6）。
- **order, period, sample_number, sample_frequency, traj_cycle** 从 config 读，缺省用 5、10、15、200、1。
- **init_joint, upper_joint, lower_joint, max_vel, max_acc** 从 config 取并保证为列向量、长度 dim。
- **“收紧可行域”**（为优化更容易满足约束）：
  - 位置：中点 center = (upper+lower)/2，半范围 half_range = (upper-lower)/2；新的上下限取 **center ± 0.4×half_range**。
  - 速度：**max_vel = 0.9×max_vel**。
  - 加速度：沿用 config.max_acceleration（不做扭矩压加速度）。
  - **init_joint** 设为 **center**（轨迹常值项与起点/终点一致）。
  - 这些收紧后的限位写回 config，供后续约束与校验使用。
- **coeff_lower / coeff_upper**：默认 -0.5 / 0.5，可从 config 覆盖。
- **COL** = dim×(2×order+1)（系数总数）；**CN** = round(period×sample_frequency)（单周期点数）。
- 打印：`开始生成（dim=..., order=..., period=..., CN=...）`。

### 3.3 确定系数 coeff（四条分支）

**分支 A：用户给了 opts.coeff**  
- 直接 **coeff = opts.coeff**，**trajPara = coeff**，不做优化。

**分支 B：config 里已有 traj_parameter**  
- **coeff = config.traj_parameter**，**trajPara = coeff**，不做优化。

**分支 C：opts.optimize == true（系数优化）**  
- **best_cond = inf**，**best_coeff = []**。
- 对 **retry = 1 : opts.max_retries**：
  - **初值 coeff0**：在 [-0.1, 0.1] 随机；**coeff0(1:dim) = init_joint**（常数项固定）。
  - 对每个关节 j，令其 cos 系数中第 order 个 **b_j(order) = -sum(b_j(1:order-1))**，使 q_j(0)=c_j+sum(b_j)=init_joint(j)。
  - 调用 **[coeff_r, cond_r] = optimize_fourier_coeff(coeff0, ...)**（见下一节）。  
    若传入 **max_iter_optim / max_feval_optim**，会在 optimize_fourier_coeff 内覆盖默认 150/5000。
  - 若 cond_r < best_cond 且 cond_r < 1000，则 **best_cond = cond_r**，**best_coeff = coeff_r**。
- 若 best_coeff 非空，**coeff = best_coeff**；否则用最后一轮 **coeff_r** 并打 warning。
- **trajPara = coeff**。

**分支 D：否则（不优化且无给定系数）**  
- 在 [coeff_lower, coeff_upper] 随机；常数项 = init_joint；每关节 cos 系数同样满足 sum(b_j)=0，使 q_j(0)=init_joint(j)。**trajPara = coeff**。

### 3.4 优化细节：optimize_fourier_coeff

- **优化变量**：长度为 COL 的系数向量 c；**常数项 c(1:dim) 的 lb/ub 都设为 init_joint**（固定）。
- **目标函数 objective(c)**：
  - 用 **evaluate_fourier_trajectory(coeff, t_obj, ...)** 在 **t_obj** 上算 refPos, refVel, refAcc。  
    有 cond_fun 时 **t_obj** 为 25 Hz 离散（或 max_iter_optim 对应更少点），以加速。
  - 若有 **cond_fun**：**f = cond_fun(refPos, refVel, refAcc)**（即 cond(W_min)）。
  - 若无：**f = -[ sum(var(refPos)) + sum(var(refVel)) + sum(var(refAcc)) ]**（越大越激励强）。
- **约束 nonlcon(c)**（全部为不等式 c≤0）：
  - **幅值约束**：对每关节 j，sum(|a_jk|)+sum(|b_jk|) − A_max_j ≤ 0，A_max_j = alpha×(upper_j−lower_j)/2。
  - **离散点约束**：在 **t_constraint**（25 Hz 离散）上算 refPos, refVel, refAcc；每点每关节：  
    q−upper≤0，lower−q≤0；|qd|−max_vel≤0；|qdd|−max_acc≤0。
  - 若 **enable_collision_check** 且存在 **robot_limb**：每点 checkCollision，碰撞则追加违反量。
- **fmincon**：lb/ub 为 coeff_lower/coeff_upper（常数项已固定）；**MaxIterations=MAX_ITER**，**MaxFunctionEvaluations=MAX_FEVAL**（可由 max_iter_optim / max_feval_optim 覆盖）；ConstraintTolerance=1e-5。
- 返回 **coeff_out**（最优或异常时初值）、**cond_val**（最终目标值）。

### 3.5 傅里叶轨迹公式与求值

- **公式**（与文档一致）：  
  - 角频率 **ω = 2π/period**。  
  - 关节 j 的位置：  
    **q_j(t) = c_j + Σ_{k=1}^{order} [ a_jk sin(kωt) + b_jk cos(kωt) ]**  
    其中 **c_j = coeff(j)**（常数项）。  
  - 速度、加速度由对 t 求一阶、二阶导得到。
- **系数在向量中的顺序**（COL = dim×(2×order+1)）：  
  - 前 dim 个：c_1..c_dim（常数项）；  
  - 接着 dim×order 个：a_11..a_1order, a_21..a_2order, ...；  
  - 再 dim×order 个：b_11..b_1order, b_21..b_2order, ...。
- **evaluate_fourier_trajectory(coeff, t, dim, order, period, init_joint)**：对给定时间向量 t（N×1），按上式逐关节、逐 k 累加，得到 **pos, vel, acc**（N×dim）。  
  用于：优化中的目标/约束、单周期 refPos/refVel/refAcc、最终 trajectory 的周期段。

### 3.6 单周期 refPos, refVel, refAcc 与条件数打印

- **t_period** = linspace(0, period, CN)'，端点保证 ≤ period。
- **[refPos, refVel, refAcc] = evaluate_fourier_trajectory(coeff, t_period, dim, order, period, init_joint)**，得到 CN×dim。
- 若提供了 **cond_fun**，再算一次 **final_cond = cond_fun(refPos, refVel, refAcc)** 并打印“生成轨迹条件数 = ...”。

### 3.7 约束校验 validate_trajectory

- 若 **opts.validate** 为 true，调用 **[ok, msg, violation_details] = validate_trajectory(refPos, refVel, refAcc, config, 1, 1, 1)**（不放宽）。
- **校验内容**（按点、按关节）：  
  - 位置：q 在 [lower, upper] 内（允许 1e-4 rad 数值容差）；  
  - 速度：|qd| ≤ max_vel；  
  - 加速度：|qdd| ≤ max_acc；  
  - 若 enable_collision_check 且存在 robot_limb：checkCollision；  
  - 若存在 robot_limb 与 max_effort：inverseDynamics 算 τ，|τ_j| ≤ max_effort(j)。
- 任一处超限即 **ok=false**，**violation_details** 记录第一条超限信息（sample_idx, joint_idx, type, value, limit, excess），并 return。
- **valid_ok = ok**；violation_details 返回给上层用于打印与重试。

### 3.8 完整 trajectory 的拼接

- **过渡段**：时长 **t_trans = 1 s**，点数 n_trans = round(t_trans×sample_frequency)。  
  - 上升段：从 **init_joint** 线性插值到 **refPos(1,:)**；  
  - 下降段：从 **refPos(end,:)** 线性插值回 **init_joint**。
- **周期段**：**refPos_rep = repmat(refPos, traj_cycle, 1)**（同理 refVel_rep, refAcc_rep）；时间轴 t_mid 按 traj_cycle 个周期铺开。
- **完整时间**：**t_full = [上升段时间; 每个周期的时间; 下降段时间(去掉首点)]**。  
- **完整位置**：**q_full = [q_ramp_up; refPos_rep; q_ramp_down(2:end,:)]**。  
- **trajectory = [t_full, q_full]**，即每行 [t, q1..q_dim]（仅左腿 6 维）。

### 3.9 输出

- **trajectory**：完整 [t, q1..q6]；**trajPara**：系数；**refPos, refVel, refAcc**：单周期 CN×dim；**t_period**：单周期时间。
- 若 nargout≥9 且 output_design_matrix，可再算 A_pos, A_vel, A_acc（辨识用设计矩阵）。
- **valid_ok**、**violation_details** 为校验结果。

---

## 四、数据流小结

1. **config** 决定：关节数 dim、傅里叶 order/period、采样 CN、限位与可行域（收紧后）、是否碰撞/扭矩校验。
2. **cond_fun** 由 ReMatrix（E1_limb）和 config 构造，用于优化目标 **cond(W_min)**。
3. **系数** 要么给定/config，要么通过 **optimize_fourier_coeff**（fmincon + 目标 cond_fun + 幅值/位置/速度/加速度/可选碰撞约束）得到，要么随机但满足 q(0)=init_joint。
4. **evaluate_fourier_trajectory** 用同一套傅里叶公式在任意 t 上得到 q, qd, qdd。
5. **单周期** 在 t_period 上求值得到 refPos, refVel, refAcc；**校验** 在这些点上做位置/速度/加速度/碰撞/扭矩检查。
6. **完整轨迹** = 1s 上升 + traj_cycle 个周期 + 1s 下降；**standalone** 再扩展为 12 关节（右腿 0）并绘图。

按上述顺序执行即“激励轨迹的完整生成流程”。

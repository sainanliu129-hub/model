# 激励轨迹优化：原理与每一步详解

本文说明「次 → 轮 → 迭代」三层结构下，**每一步在做什么、为什么这样做**，以及用到的数学与代码对应关系。

---

## 一、总目标与整体流程

**目标**：找到一组傅里叶系数 **coeff**，使得由该系数生成的轨迹  
q(t), qd(t), qdd(t) 满足：

1. **约束**：位置/速度/加速度/扭矩不超限，幅值在安全范围内（可选自碰撞）。
2. **最优**：在该约束下，轨迹对应的辨识矩阵 **W** 的条件数 **κ(W)** 尽量小（辨识效果更好）。

**实现方式**：用数值优化器（fmincon）**最小化** 目标 **f(coeff) = κ(W)**，同时满足一系列**不等式约束**。  
因为初值随机、约束非凸，所以用「多轮不同初值 + 多次尝试校验」提高得到可用轨迹的概率。

**三层结构**：

- **次**：一整次「多轮优化 → 得到系数 → 生成轨迹 → 校验」。校验不过就再跑下一次。
- **轮**：在一次里，用**不同随机初值**跑多轮优化，每轮调用一次 fmincon，取「exitflag>0 且 cond<1000」的最好系数。
- **迭代 (iter)**：在**某一轮**里，fmincon 的**一步一步**：每步更新系数、算目标与约束、判断是否停止。

下面按「次 → 轮 → 迭代」从外到内说明每一步在干什么。

---

## 二、一次「次」(try_i) 里发生了什么

**位置**：`run_excitation_trajectory_standalone.m` 中 `for try_i = 1:max_tries`。

**含义**：一次「次」= 一次完整的「生成轨迹并校验」流程。

**具体步骤**：

1. **调用** `generate_excitation_trajectory(config, ...)`  
   内部会做：
   - 多**轮**优化（见第三节），得到当前认为最好的系数 **coeff**；
   - 用该系数按傅里叶公式生成单周期 **refPos, refVel, refAcc**；
   - 用这些轨迹拼出完整 **trajectory**（过渡段 + 周期重复 + 回零）；
   - 对单周期轨迹做 **validate_trajectory**（位置/速度/加速度/扭矩等），得到 **valid_ok**。

2. **判断**  
   - 若 **valid_ok == true**：本「次」成功，跳出循环，用当前 trajectory 做后续绘图与输出。  
   - 若 **valid_ok == false**：本「次」失败（轨迹超限），打印 violation_details；若 try_i < max_tries，则 **try_i+1** 再跑一次「次」（重新做多轮优化 + 生成 + 校验）。

**为什么要有「次」**：优化只在**离散点**上满足约束，且校验时采样更密（或不同点），可能在某些点上超限。多「次」等于多给几次「重新优化 + 校验」的机会，提高最终通过校验的概率。

---

## 三、一轮「轮」(retry) 里发生了什么

**位置**：`generate_excitation_trajectory.m` 中 `for retry = 1:opts.max_retries`（在某一「次」之内）。

**含义**：一轮 = 用一个**新的随机初值** coeff0，调用一次 **optimize_fourier_coeff**（即跑一遍 fmincon），得到一组系数与条件数。

**具体步骤**：

1. **生成随机初值 coeff0**  
   - 在 [-0.1, 0.1] 上随机；**coeff0(1:dim)** 设为 **init_joint**（常数项 = 轨迹起点/终点）。  
   - 对每个关节 j，令 cos 系数的第 order 个 **b_j(order) = -sum(b_j(1:order-1))**，保证 q_j(0) = c_j + Σb_j = init_joint(j)，即轨迹从 init_joint 出发。

2. **调用** `[coeff_r, cond_r, exitflag] = optimize_fourier_coeff(coeff0, ...)`  
   即用 **coeff0** 作为初值，跑**一整次** fmincon（内部会做很多**迭代**步），得到：
   - **coeff_r**：该轮优化得到的系数；
   - **cond_r**：该系数对应的目标值（条件数 κ(W) 或激励量代理）；
   - **exitflag**：优化器退出原因（>0 正常，≤0 未收敛或异常）。

3. **是否接受该轮结果**  
   仅当 **exitflag > 0** 且 **cond_r < CALIB_TRAJECTORY_MAX_COND_NUM(1000)** 时，才和当前 best_cond 比较；若 cond_r 更小，则 **best_coeff = coeff_r**，**best_cond = cond_r**。

4. **多轮结束后**  
   若存在 **best_coeff**（至少有一轮「优化成功」），则本「次」用的系数为 **best_coeff**；否则用最后一轮的 **coeff_r**（并打 warning）。

**为什么要有「轮」**：优化是**非凸**的，不同初值可能收敛到不同局部最优（或不同可行解）。多轮随机初值可以多试几条「下山路径」，取其中条件数最好且满足 exitflag>0、cond<1000 的那组系数。

---

## 四、一次「迭代」(iter) 里发生了什么

**位置**：在 **optimize_fourier_coeff** 内部，由 **fmincon** 执行；每一 iter 对应 fmincon 的一次迭代步。

**含义**：在**同一轮、同一初值**下，优化器从当前系数 **x** 出发，算目标与约束、决定搜索方向与步长、更新 **x**，并判断是否停止。

**数学形式**：

- **决策变量**：**x = coeff**（维数 COL = dim×(2×order+1)），其中 **x(1:dim)** 被 lb/ub 固定为 **init_joint**，实际在动的是后面的 sin/cos 系数。
- **目标函数**：**f(x) = κ(W)**。  
  - 先用 **x** 按傅里叶公式在时间点 **t_obj**（如 12 Hz 离散）上算出 **refPos, refVel, refAcc**；  
  - 再用 **cond_fun(refPos, refVel, refAcc)**：建观测矩阵 W → 列归一化 → rref 取独立列得 W_min → **κ = sqrt(cond(W_min'*W_min))**；  
  - fmincon 是**最小化** f。
- **约束**：全部为不等式 **c(x) ≤ 0**（无等式）。包括：
  - 幅值：对每关节 j，**sum(|a_jk|)+sum(|b_jk|) − A_max_j ≤ 0**；
  - 离散点上的位置：**q − upper ≤ 0**，**lower − q ≤ 0**；
  - 离散点上的速度：**|qd| − max_vel ≤ 0**；
  - 离散点上的加速度：**|qdd| − max_acc ≤ 0**；
  - 若有 robot_limb 与 max_effort：**|τ| − max_effort ≤ 0**；
  - 若开启碰撞：碰撞则追加违反量。

**单次迭代 (iter) 里 fmincon 实际做的事**（简化说明）：

1. **当前点**：**x** = 当前系数。
2. **算目标**：**f(x)** = eval_objective(x) → 由 x 生成轨迹 → 算 κ(W)。
3. **算约束**：**c(x)** = eval_constraints(x) → 由 x 在 **t_constraint**（如 12 Hz）上生成轨迹 → 算所有不等式违反量（位置/速度/加速度/幅值/扭矩/碰撞等）。
4. **算梯度信息**：fmincon 用数值差分或内部方式得到目标与约束关于 x 的梯度信息，用于构造搜索方向。
5. **搜索方向与步长**：在满足约束或惩罚约束的前提下，决定沿哪个方向、走多大步长（一维搜索或信赖域等）。
6. **更新**：**x_new = x + 步长 × 方向**（并投影到 lb/ub 内）。
7. **停止判断**：若满足以下**任一**则本轮结束、返回：
   - **f(x) ≤ STOPVAL(113)**（OutputFcn 检测到即停）；
   - 目标绝对变化 < **FunctionTolerance(1e-1)**；
   - 步长/系数变化 < **StepTolerance(1e-5)**；
   - 一阶最优性条件足够好；
   - 迭代次数达到 **MaxIterations**；
   - 函数评估次数达到 **MaxFunctionEvaluations**。

**为什么需要很多「迭代」**：κ(W) 与约束都是系数 **x** 的非线性、非凸函数，无法一步算到最优。只能从当前 x 一步步下降 f、同时满足 c≤0，直到收敛或达到步数/评估上限。

## 五、公式与代码对应（便于对照）

### 5.1 傅里叶轨迹

- **公式**：  
  **q_j(t) = c_j + Σ_{k=1}^{order} [ a_jk sin(kωt) + b_jk cos(kωt) ]**，  
  ω = 2π/period；qd、qdd 为对 t 求一阶、二阶导。
- **代码**：`evaluate_fourier_trajectory(coeff, t, dim, order, period, init_joint)`；**coeff** 中前 dim 个为 c_j，随后为 a_jk、b_jk（顺序与 C++ 一致：dim×(2×order+1)）。

### 5.2 目标 κ(W)

- **含义**：W 为观测矩阵（每行对应某时刻某关节的动力学回归项），列归一化后 rref 取独立列得 W_min，**κ(W) = sqrt(cond(W_min'*W_min))**。
- **代码**：`cond_fun(refPos, refVel, refAcc)` → `cond_fun_from_regressor` → ReMatrix 得到 W → 列归一化 → rref → W_min → **sqrt(cond(W_min'*W_min))**。

### 5.3 约束

- **幅值**：对每关节 j，**Σ_k(|a_jk|+|b_jk|) ≤ A_max_j**，A_max_j = alpha×(upper_j−lower_j)/2。  
  代码：eval_constraints 里按系数索引累加 abs，与 A_max_j 比较。
- **位置/速度/加速度**：在 **t_constraint** 上算 q,qd,qdd，要求 q∈[lower,upper]，|qd|≤max_vel，|qdd|≤max_acc。  
  代码：eval_constraints 中 for i = 1:size(refPos,1) 逐点堆叠 c。
- **扭矩**：在相同离散点上用 **inverseDynamics(robot_limb, q, qd, qdd)** 算 τ，要求 **|τ_j| ≤ max_effort(j)**。  
  代码：eval_constraints 末尾若存在 robot_limb 与 max_effort 则追加 **abs(tau)−max_effort**。

## 六、数据流小结（谁调用谁、谁用谁的结果）

```
run_excitation_trajectory_standalone
  for try_i = 1:max_tries   【次】
    → generate_excitation_trajectory
        for retry = 1:max_retries   【轮】
          生成随机 coeff0
          → optimize_fourier_coeff(coeff0, ...)   【整轮 = 一次 fmincon】
              构造 objective = @(c) eval_objective(...)  → 算 κ(W)
              构造 nonlcon   = @(c) eval_constraints(...) → 算 c≤0
              fmincon(objective, coeff0, ..., nonlcon, options)
                 iter 0: 算 f(x0), c(x0)
                 iter 1: 方向、步长，更新 x；算 f(x1), c(x1)
                 ...
                 直到停止 → 返回 (coeff_out, fval, exitflag)
          ← (coeff_r, cond_r, exitflag)
          若 exitflag>0 且 cond_r<1000：视情况更新 best_coeff
        ← best_coeff 或最后一轮 coeff_r
        用 coeff 生成 refPos, refVel, refAcc 和 trajectory
        validate_trajectory(refPos, refVel, refAcc, ...) → valid_ok
    ← (trajectory, ..., valid_ok)
    若 valid_ok 则 break；否则 try_i+1 再跑一次「次」
```

## 七、小结表

| 层级 | 循环/位置 | 单步在做什么 | 停止/结束条件 |
|------|-----------|--------------|----------------|
| **次** | try_i, max_tries | 整次「多轮优化→取 best 系数→生成轨迹→校验」 | 校验通过即停；否则重试，最多 max_tries 次 |
| **轮** | retry, max_retries | 新随机初值 coeff0 → 调用一次 fmincon → 得到 (coeff_r, cond_r, exitflag) | 跑满 max_retries 轮；接受条件：exitflag>0 且 cond<1000 |
| **迭代** | fmincon 内部 | 当前系数 x → 算 f(x)、c(x) → 方向与步长 → 更新 x → 判停 | stopval/ftol/xtol/最优性/MaxIter/MaxFunEval 任一满足即停 |

这样，「次」管整次生成与校验是否通过，「轮」管多初值多跑几遍优化，「迭代」管在某一初值下 fmincon 一步一步把条件数压下去并满足约束。

# 单电机模块转动惯量测量说明

适用于**单电机、末端无负载**，在上位机 Python 采集 `timestamp_s, direction, target_speed_rpm, command_current_A, actual_speed_rpm, phase_current_A, temp_degC` 的场景。  
在完成**静摩擦**与**库伦/粘滞摩擦**辨识后，按下面方法测**转动惯量**。

---

## 怎么调用（简要）

| 步骤 | 脚本 / 函数 | 说明 |
|------|-------------|------|
| 一次完成 | `run_full_iden_three_files` | 改 `file_static`、`file_velocity`、`file_accel`、`K_tau` 后运行，得到 τ_s、τ_c、b、I_a |
| 仅惯量（已有 τ_c、b） | `identify_inertia_from_data(文件, K_tau, tau_c, b, 'DataFormat','python_upper', ...)` | 加速度扫描 6/7 列 CSV |

**推荐**：用 Python 电流/速度/加速度三路扫描得到三个 CSV，运行 `run_full_iden_three_files.m` 一次得到 τ_s、τ_c、b、I_a。

---

## 一、原理

动力学关系（单轴、无负载）：

- **τ_real = I_a × q̈ + τ_friction**
- 净力矩：**τ_net = τ_real − τ_friction = I_a × q̈**
- 故：**I_a = τ_net / q̈**

因此需要一段**有加速度**的轨迹（不是匀速）：测出**时间、电流（或力矩）、角速度**，由速度差分得到角加速度 q̈，再用已辨识的摩擦算出 τ_friction，即可拟合 I_a。

**注意**：**速度扫描文件**（多档匀速）不能用来辨识 I_a。匀速段 q̈≈0，没有有效的加速度信号，`identify_inertia_from_data` 会报“有效加速度数据点不足”。I_a 必须用**恒流加速**、**速度斜坡**或**恒定加速度轨迹**的实测数据。

**速度斜坡可以**：电机不能直接给加速度时，给**速度斜坡**（目标速度线性增加，如 ω_target = k·t）即可。斜坡段内实际角加速度 q̈ 近似常数，与“恒定加速度”等价，辨识流程和调用方式完全相同，只需导出 t、电流、actual_speed_rpm 的 CSV 即可。

---

## 二、推荐测试方式

### 方式 A：恒流加速（或恒力矩加速）

思路：给电机**恒定电流**（或恒定力矩），电机从静止或低速加速，记录 **t、ω、i**，用速度差分得到 q̈。

### 2.1 操作步骤

1. **恒流加速**
   - 设定一个**恒定电流**（如 0.5A 或 1A，保证加速明显且不过载），方向固定（正或负）。
   - 电机从静止开始加速，运行数秒（例如 3～5 s），采样率建议 ≥200 Hz（与您现有 Python 采集一致即可）。
   - 记录：`timestamp_s`, `actual_speed_rpm`, `phase_current_A`（以及需要的 direction 等）。

2. **可选：正反各做一次**
   - 正向恒流加速一段，再反向恒流加速一段，可得到正负加速度点，拟合更稳。

3. **导出为惯量辨识用 CSV**
   - 至少三列：**时间(s)、力矩(N·m)或电流(A)、角速度(rpm)**。  
   - 若列是电流，MATLAB 端用 `K_tau` 转成力矩，并设 `'tau_is_current', true`。

### 方式 B：速度斜坡（电机不能直接给加速度时）

思路：上位机给**目标速度线性增加**（速度斜坡），例如从 0  ramp 到某值再 ramp 回 0。斜坡段内 **q̇ ∝ t**，故 **q̈ = 常数**，与恒定加速度等价。

1. **速度斜坡**
   - 目标速度：ω_target(t) = k·t（或分段斜坡：0→v_max→0），斜率 k 不要过大以免过载。
   - 记录与恒流加速相同：`timestamp_s`, `actual_speed_rpm`, `phase_current_A`（6/7 列格式即可）。
2. **数据处理**
   - 由 actual_speed_rpm 对时间差分得到 q̈；斜坡段 |q̈| 明显非零，可直接用于惯量辨识。
3. **调用方式**
   - 与恒流加速**完全一致**：同一 CSV 导出格式；三文件流程用 `run_full_iden_three_files.m`，或单独惯量时调用 `identify_inertia_from_data(..., 'DataFormat','python_upper')`。

只要轨迹中有**足够长的加速/减速段**（恒流加速或速度斜坡均可），就能得到可靠的 I_a。

### 2.2 与您当前 Python 数据的对应关系

您现有列：`timestamp_s`, `direction`, `target_speed_rpm`, `command_current_A`, `actual_speed_rpm`, `phase_current_A`, `temp_degC`。

惯量辨识只需其中三列即可：

| 本仓库 3 列格式 | 您的 Python 列 | 说明 |
|-----------------|----------------|------|
| 第 1 列：时间 (s) | `timestamp_s` | 相对时间或绝对时间均可，能算 dt 即可 |
| 第 2 列：力矩或电流 | `phase_current_A` | 若用电流，MATLAB 里用 K_tau 转换并设 tau_is_current=true |
| 第 3 列：角速度 (rpm) | `actual_speed_rpm` | 保持 rpm；脚本内可转 rad/s |

导出示例（仅三列，无表头或表头会被脚本跳过）：

```text
timestamp_s,phase_current_A,actual_speed_rpm
1770952414.422,0.50,-0.001
1770952414.424,0.50,0.12
1770952414.426,0.50,0.28
...
```

或带表头的一行说明（脚本会跳过导致 NaN 的表头行）：

```text
time_s,current_A,speed_rpm
1770952414.422,0.50,-0.001
...
```

## 三、在 MATLAB 中辨识 I_a

前提：已做过摩擦辨识，得到 **τ_c（或 τ_c_pos/τ_c_neg）、b**，以及力矩系数 **K_tau**（若用电流则必须）。

### 3.1 三列格式（时间、电流、速度 rpm）

```matlab
% 先完成摩擦辨识，得到 Result_friction
% Result_friction = identify_friction_from_trajectory(...);

% 惯量辨识：3 列 = time, current, velocity_rpm
Result_inertia = identify_inertia_from_data(...
    'accel_constant_current_001.csv', ...   % 恒流加速采集的 CSV
    K_tau, ...
    Result_friction.tau_c, ...
    Result_friction.b, ...
    'tau_is_current', true, ...             % 第 2 列为电流
    'VelocityUnit', 'rpm', ...              % 第 3 列为 rpm
    'DoPlot', true);
```

若您用**不对称库伦摩擦**（τ_c_pos / τ_c_neg）：

```matlab
Result_inertia = identify_inertia_from_data(...
    'accel_constant_current_001.csv', ...
    K_tau, ...
    Result_friction.tau_c, ...   % 可传对称值，下面会覆盖
    Result_friction.b, ...
    'tau_is_current', true, ...
    'VelocityUnit', 'rpm', ...
    'TauCPos', Result_friction.tau_c_pos, ...
    'TauCNeg', Result_friction.tau_c_neg);
```

### 3.2 脚本在内部做了什么

1. 读入 3 列：t, 电流(或力矩), 角速度(rpm)。
2. 将 rpm 转为 rad/s（若指定 `VelocityUnit','rpm'`）。
3. 用**速度数值微分**得到 q̈，再做滑动平均滤波。
4. τ_real = K_tau × 电流（当 `tau_is_current=true`）。
5. τ_friction 用您已辨识的 τ_c、b（及可选 τ_c_pos/τ_c_neg）计算。
6. 线性回归：**τ_net = I_a × q̈**，得到 **I_a** 及 R²、RMSE。

因此您只要保证 CSV 里在**加速段**有足够的 (t, current, speed_rpm) 即可。

---

## 四、数据与采样建议

- **采样率**：与摩擦辨识一致或更高（如 200～500 Hz），便于速度差分得到平滑的 q̈。
- **加速时间**：2～5 s 一般足够；加速度越大，|q̈| 越大，拟合越稳（脚本会过滤 |q̈| 过小的点，见 `accel_threshold`）。
- **电流大小**：在安全范围内尽量让加速明显（例如 0.3～1 A 视电机而定），避免长时间接近匀速（q̈≈0）。
- **滤波**：脚本内部对 q̈ 做了 `movmean` 滤波，若噪声大可在 Python 端对 `actual_speed_rpm` 先做轻滤波再导出，或保持高采样率。

---

## 五、小结

| 项目 | 内容 |
|------|------|
| **物理** | I_a = (τ_real − τ_friction) / q̈，需要**有加速度**的区段 |
| **能否用速度扫描？** | **不能**。速度扫描是匀速，q̈≈0，无法拟合 I_a |
| **测试** | 恒流（或恒力矩）加速，记录 t、电流、actual_speed_rpm |
| **导出** | CSV 至少 3 列：时间(s)、电流(A)、角速度(rpm) |
| **MATLAB** | `identify_inertia_from_data(..., 'tau_is_current', true, 'VelocityUnit', 'rpm')`，并传入已辨识的 τ_c、b 和 K_tau |

按上述方式在上位机做一次“恒流加速”采集，导出三列 CSV，即可在现有流程下完成转动惯量测量与辨识。

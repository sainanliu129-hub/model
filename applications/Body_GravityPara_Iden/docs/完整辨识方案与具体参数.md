# 完整辨识方案与具体参数

本文档给出 **摩擦/惯量辨识（Friction_Iden）→ 刚体/重力参数辨识（Body_GravityPara_Iden）→ FD 验证** 的端到端方案及可直接使用的具体参数。

---

## 一、整体流程顺序

| 步骤 | 目录 | 脚本/操作 | 输出 |
|------|------|-----------|------|
| 1 | Friction_Iden | 采集 3 类 CSV + `run_full_iden_three_files` | τ_s、τ_c、b、I_a → `全部电机参数汇总.xlsx` |
| 2 | Body_GravityPara_Iden | 采集激励 CSV，`run_min_param_id_from_csv`（默认带预处理） | X_hat、index_base、prep_opts → `min_param_id_result.mat` |
| 3 | Body_GravityPara_Iden | `run_full_dynamics_validation`（或 `run_debug_pi_phys_id_fd_plot` 调 π_fd） | ID/FD 对比图与误差统计 |

**依赖**：步骤 2 建议先完成步骤 1，并在步骤 2 中开启 `load_friction_from_summary = true`，从汇总表读摩擦与 I_a，再做刚体辨识。

---

## 二、摩擦与惯量辨识（Friction_Iden）

### 2.1 数据采集：三类 CSV

| 类型 | 用途 | 说明 |
|------|------|------|
| 电流扫描 CSV | 静摩擦 τ_s | 关节保持不动，电流阶跃往返；从静止段 max\|i\| 得 τ_s ≈ K_τ·max\|i\| |
| 速度扫描 CSV | 动摩擦 τ_c、b | 缓慢加减速/锯齿速度，覆盖工作速度；匀速段 (q̇, τ) 拟合 τ_f(q̇) |
| 加速度扫描 CSV | 惯量 I_a | 分段定频正弦，良好跟踪；记录 q、q̇、i，用于 τ_net = τ_real − τ_f 与 q̈ 拟合 I_a |

### 2.2 轨迹参数（通用、与摩擦测试方案一致）

| 参数 | 取值 | 说明 |
|------|------|------|
| 分段正弦（加速度扫描） | | |
| 幅值 A | **0.5 rad** | 可依关节行程缩小，如 min(0.5, 0.3×行程) |
| 频率范围 | **0.1 Hz ~ 2.0 Hz** | f_start=0.1, f_end=2.0，线性递增 |
| 段数 × 每段时长 | **10 段 × 6 s，总 60 s** | NumSegments=10, SegmentDuration=6 |
| 位置限幅 | \|q\| ≤ **12.5** rad | 公共上限 |
| 速度限幅 | \|q̇\| ≤ **30** rad/s | 公共上限 |
| 控制增益 | **Kp=20, Kd=2** | 可统一，必要时单机微调 |

### 2.3 run_full_iden_three_files 配置与辨识参数

**脚本路径**：`applications/Friction_Iden/run_full_iden_three_files.m`

**需修改的配置（脚本顶部）**：

| 变量 | 说明 | 示例 |
|------|------|------|
| `dd` | 数据根目录 | `get_data_dir('test_data/因克斯EC-A10020-P2-72V')` |
| `file_static` | 电流扫描 CSV | `fullfile(dd, 'scan_current_serial_20260310_164327.csv')` |
| `file_velocity` | 速度扫描 CSV | `fullfile(dd, 'scan_result_20260310_170140.csv')` |
| `file_accel` | 加速度扫描 CSV | `fullfile(dd, 'scan_accel_serial_mit_20260311_160046.csv')` |
| `file_accel_is_torque` | 加速度文件第 2 列是否为力矩 | `false`（电流） / `true`（力矩） |
| `K_tau` | 力矩系数 τ = K_τ × i | `2.6`（按电机标定） |
| `FrictionModel` | 摩擦模型 | `'tanh_viscous'` / `'coulomb_viscous'` / `'stribeck_viscous'` |
| `Fs` | 采样频率 (Hz) | `200` |

**固定辨识参数（通用方案，一般不需改）**：

| 参数 | 取值 | 说明 |
|------|------|------|
| VarianceSpeedPct | **5** | 匀速段判定：std(q̇) ≤ 此比例 × \|q̇\| |
| R2Min | **0.5** | 段有效阈值（R² 大于此才参与 I_a 汇总） |
| AccelMeanMin | **0** | 不按 mean(\|q̈\|) 再筛 |
| MiddleFrac | **0.6** | 每段取中间 60% |
| LowpassCutoffHz | **12** | 速度低通 (Hz) |
| JMethod | **weighted_mean** | 按 R² 加权汇总 I_a |
| SegmentDuration / NumSegments | **6 s / 10** | 与 60 s、10 段轨迹一致 |
| speed_still_rads | **0.05×(2π/60)** | 静止判定阈值 (rad/s) |
| stable_start / stable_end | **100 / 20** | 静摩擦稳定段首尾去掉点数 |
| accel_threshold | **0** | \|q̈\| 下限，0=用全部点 |
| accel_exclude_above | **500** | 剔除 \|q̈\| > 此值的点 |
| tau_exclude_above | **500** | 剔除 \|τ\| > 此值的点 |

**运行**：在 MATLAB 中 `cd` 到 `applications/Friction_Iden`，运行 `run_full_iden_three_files`。

**输出**：`build/全部电机参数汇总.xlsx`、单电机 Excel 与报告、PNG 图。

## 三、刚体/重力参数辨识（Body_GravityPara_Iden）

### 3.1 数据来源

- **激励 CSV**：多关节激励轨迹采集（格式与 `read_leg_joint_csv` 兼容），通常放在 `data/excitation/` 或 `data/跑步/`。
- **摩擦/转子**：从 Friction_Iden 的 `build/全部电机参数汇总.xlsx` 读取（推荐），在刚体辨识脚本中设 `load_friction_from_summary = true`。

### 3.2 预处理参数（与 run_torque_comparison 一致）

以下为 **run_min_param_id_from_csv** 默认采用的 `prep_opts`（`use_preprocess_id = true` 时），与 **run_torque_comparison** 完全一致。

| 参数 | 取值 | 说明 |
|------|------|------|
| **时间窗** | | |
| t_start_s | **2.1** | 有效段起始时间 (s) |
| t_end_s | **4.1** | 有效段结束时间 (s) |
| **Savitzky-Golay（由 q 得 q/q̇/q̈）** | | |
| sg_order | **4** | 多项式阶次 |
| sg_frame | **23** | 窗长（奇数），500 Hz 约 0.046 s |
| **力矩低通** | | |
| tau_lowpass_fc_Hz | **15** | Butterworth 截止频率 (Hz) |
| tau_lowpass_order | **2** | Butterworth 阶数 |
| **补偿项（τ_id = τ_s − J_eq·q̈ − τ_f − τ_bias）** | | |
| J_eq | 从汇总表 I_a 来 | 转子惯量 1×6，`load_friction_from_summary` 时自动填 |
| friction_params | 从汇总表来 | Stribeck/粘滞等，同上 |
| tau_bias | 默认 0 | 常值偏置（可选） |
| **其它** | | |
| do_plot | **false** | 预处理每步是否画图 |
| plot_joints | **[1 2 3]** | 参与绘图的关节 |

**连续窗（预处理后）**：不再二次裁时间、不重算 q̈，`win_opts = struct('t_start_s', [], 't_end_s', [], 'qd_lowpass_fc_Hz', 0, 'qdd_smooth_half', 0)`。

### 3.3 run_min_param_id_from_csv 配置与默认参数

**脚本路径**：`applications/Body_GravityPara_Iden/run_min_param_id_from_csv.m`（脚本，直接点运行）

**配置区（文件开头）**：

| 变量 | 默认/说明 | 常用修改 |
|------|-----------|----------|
| csv_file | **[]** | 留空则自动用 `data/excitation/` 或 `data/跑步/` 下第一个 CSV；或设为 `'data/excitation/你的文件.csv'` |
| opts | **struct()** | 见下表 |

**opts 默认与可选字段**：

| 字段 | 默认值 | 说明 |
|------|--------|------|
| mode | **'continuous'** | 连续窗辨识；'cycle_average' 为周期平均 |
| use_preprocess_id | **true** | 与 run_torque_comparison 一致预处理 |
| prep_opts | **struct()** | 不设则用上节表格中的默认（t_start_s=2.1, t_end_s=4.1, sg_frame=23 等） |
| load_friction_from_summary | **false** | 建议改为 **true**，从汇总表读摩擦与 I_a |
| n_joints | 6 | 关节数 |
| row_for_joint | [] | 关节→汇总表行映射，缺省自动 |
| continuous_opts.t_start_s | 2.1 | 与 prep 一致（预处理时由 prep 裁窗，此项仅未预处理时用） |
| continuous_opts.t_end_s | 4.1 | 同上 |
| continuous_opts.qd_lowpass_fc_Hz | 0 | 预处理时窗内不再滤波 |
| continuous_opts.qdd_smooth_half | 0 | 预处理时窗内不再平滑 |
| K, T, trim_ramp, anchor_* 等 | 周期平均用 | mode='cycle_average' 时有效 |

**辨识内部**：`identify_min(..., opts_id)`，`use_wls=false`；若已预处理则不再在 identify_min 中扣摩擦/转子。

**输出**：`min_param_id_result.mat`（X_hat, index_base, avg_data, metrics, prep_opts）。

### 3.4 run_torque_comparison 参数（对比用）

与 run_min_param_id_from_csv 使用同一套预处理时，参数一致即可。若单独运行 run_torque_comparison（函数调用），推荐传入与上一致的选项：

| 参数 | 取值 |
|------|------|
| csv_file | 同激励 CSV 路径 |
| opts_in.t_start_s | 2.1 |
| opts_in.t_end_s | 4.1 |
| opts_in.load_friction_from_summary | true |
| opts_in.use_saved_id | true 可跳过辨识、仅用已保存 mat 画对比 |

内部 prep_opts：t_start_s=2.1, t_end_s=4.1, sg_order=4, sg_frame=23, tau_lowpass_fc_Hz=15, tau_lowpass_order=2, J_eq=I_a, friction_params 从汇总表加载。

## 四、FD 验证（已收口到 run_full_dynamics_validation / run_debug_pi_phys_id_fd_plot）

### 4.1 前置

- 已运行 `run_min_param_id_from_csv` 生成 `min_param_id_result.mat`。

### 4.2 配置与参数

**脚本路径**：`applications/Body_GravityPara_Iden/test/run_full_dynamics_validation.m`（全流程含 FD 对比）  
需要单独调 π_fd 时用：`applications/Body_GravityPara_Iden/test/run_debug_pi_phys_id_fd_plot.m`

| 变量 | 默认/说明 | 说明 |
|------|-----------|------|
| use_test_trajectory | **true** | 说明：原“换轨迹 FD 验证”脚本已删除；目前建议在 `run_full_dynamics_validation` 中改 CSV/时间窗，或在 `run_debug_pi_phys_id_fd_plot` 中换数据窗口后对比 |
| csv_file_test | `data/excitation/PD-M1-v0_multi_joint_...csv` | 换轨迹时的验证 CSV |
| test_window_opts | t_start_s=**0**, t_end_s=**10**, qd_lowpass_fc_Hz=**0**, qdd_smooth_half=**0** | 换轨迹时的时间窗；若 mat 含 prep_opts，会先用同预处理再取此窗 |
| point_idx_range | **[1, 3718]** | 用于 FD/力矩对比的点的下标范围，可缩小以减耗时 |

**换轨迹且 mat 中含 prep_opts 时**：对 csv_file_test 先做 `preprocess_id_data(..., prep)`（prep 时间窗改为 test_window_opts.t_start_s / t_end_s），再 `continuous_window_id_data(..., 空窗, qdd_for_win)`，得到与辨识同预处理的 q_meas, qd_meas, tau_meas, qdd_meas。

### 4.3 运行

在 MATLAB 中 `cd` 到 `applications/Body_GravityPara_Iden/test`，运行 `run_full_dynamics_validation`（或 `run_debug_pi_phys_id_fd_plot`）。

## 五、参数速查表（复制即用）

### 5.1 Body_GravityPara_Iden 预处理与窗（统一）

```matlab
% 预处理 prep_opts（preprocess_id_data）
t_start_s         = 2.1;
t_end_s           = 4.1;
sg_order          = 4;
sg_frame          = 23;
tau_lowpass_fc_Hz = 15;
tau_lowpass_order = 2;
% J_eq、friction_params 由 load_friction_from_summary 填

% 预处理后的连续窗 win_opts（不再裁时、不滤波 qd/qdd）
win_opts = struct('t_start_s', [], 't_end_s', [], 'qd_lowpass_fc_Hz', 0, 'qdd_smooth_half', 0);
```

### 5.2 run_min_param_id_from_csv 推荐配置

```matlab
csv_file = [];   % 或 'data/excitation/你的文件.csv'
opts = struct();
opts.mode = 'continuous';
opts.use_preprocess_id = true;
opts.load_friction_from_summary = true;   % 推荐
% 其余用脚本默认即可（prep 与上表一致）
```

### 5.3 换轨迹验证（用 run_full_dynamics_validation）

```matlab
% 在 run_full_dynamics_validation.m 中：
% 1) 把 csv_file 改成你的验证文件
% 2) 把 Step 2 的 continuous_opts/prep_opts 与 fd_t_start_s/fd_t_end_s 调到你想要的时间窗
% 3) 若只想看 π_fd：用 run_debug_pi_phys_id_fd_plot.m，替换数据窗后画 FD/ID 对比
```

### 5.4 Friction_Iden 通用辨识参数（run_full_iden_three_files 内固定）

```matlab
VarianceSpeedPct = 5;
R2Min = 0.5;
MiddleFrac = 0.6;
LowpassCutoffHz = 12;
SegmentDuration = 6;
NumSegments = 10;

```

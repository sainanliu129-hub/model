## 一、核心方案

| 方法 | 核心方案 |
| ---- | -------- |
| **Panda 2019** | τ = Yπ，先在线性空间辨识 base/最小参数，再通过非线性优化在物理约束下恢复全参数（质量、CoM、惯量），最后用全参数做动力学。 |
| **Wensing 2017（LMI）** | 直接在 τ = Yπ 的辨识问题中加入惯量 LMI 约束，识别得到本身物理一致的全参数（不另设恢复阶段）。 |
| **Sensors 2025（最小参数+FD）** | 完全在最小参数空间工作：τ = Hλ，经结构分解得到 λ_min，再由 λ_min 直接构造 M(q)，绕开显式全参数。 |
| **本方案一：FD 导向全参数辨识** | 在 τ = Y(q,qd,qdd)·π 框架下，直接以全参数 π 为优化变量，同时最小化 τ 误差与 qdd_FD 误差，并在目标与约束中加入物理先验与 M(q) 正定性。 |
| **本方案二：最小参数 + 物理恢复** | 先用线性最小二乘在最小参数 β（X_hat）空间拟合 τ，再用有物理约束的非线性优化从 β 恢复物理可行全参数 π_phys，最后用 π_phys 做 FD。 |

**各方法核心公式与原理**  
- **Panda**：τ = Yπ；先 OLS 得 base/最小参数，再在物理约束下非线性恢复全参数 π，用 π 做 FD。原理：分两阶段，先线性后约束恢复。  
- **Wensing**：τ = Yπ；辨识与约束一步完成，约束为 LMI。原理：凸优化保证全局最优且 π 直接物理一致。  
- **Sensors**：τ = Hλ，H = H₁K → λ_min；用 λ_min 直接构造 M(q) 做 FD。原理：不恢复全参数，最小参数与 M(q) 线性关联。  
- **本方案一**：τ = Y·π；π 为优化变量，目标 J = w_tau‖Yπ−τ‖² + w_qdd‖qdd_FD−qdd_ref‖² + w_cad(π−π_cad)′W(π−π_cad) + w_M·J_M，硬约束 m/CoM/I。原理：同一优化里同时拟合 ID 与 FD，并惩罚 M 病态。  
- **本方案二**：τ = Y_min·β；β_hat = (Y_min\τ)，再 π_phys = argmin{‖Y_full π − Y_min β_hat‖² + λR(π,π_cad)} s.t. 物理约束。原理：先线性最小参数，再约束恢复全参数，与 Panda 两阶段类似。

**本单位方案与论文的区别**  
- **相对 Panda**：方案一不做“先 base 再恢复”，而是**直接优化全参数 π**，且目标里多了 **J_qdd、J_M**，约束用**硬约束**（fmincon nonlcon）而非纯 penalty；方案二与 Panda 思路最接近（最小参数 → 约束恢复），但最小参数用 **QR 选列** 而非解析 base，恢复目标用 **Y_full π ≈ Y_min β_hat** + CAD 正则。  
- **相对 Wensing**：本单位**不用 LMI**，用显式不等式约束（质量区间、CoM box、I 正定与三角不等式），非凸，可能局部最优；方案一还多了 FD 项与 M 正定惩罚。  
- **相对 Sensors**：本单位**不采用**“λ_min 直接构造 M”的路线；方案二若用 **forward_dynamics_min** 则形式上类似（β 直接进 FD），但 M 由 Y_min 列构造、易病态，故推荐恢复 π 后用 **forward_dynamics_full**，即仍是“全参数 → M、h → FD”。

## 二、创新点

| 方法 | 创新点 |
| ---- | ------ |
| **Panda 2019** | 系统化地从可辨识参数恢复物理参数，引入质量正、惯量正定、三角不等式、质心范围等物理约束，显式面对“τ 拟合好但参数不物理”的问题。 |
| **Wensing 2017（LMI）** | 用 LMI 形式 \(\tfrac{1}{2}\mathrm{Tr}(I)I - I \succeq 0\) 等价表达惯量物理可行性，使辨识变成凸优化 / SDP，保证全局最优且惯量严格物理一致。 |
| **Sensors 2025** | 证明质量矩阵 M(q) 也可线性表示在最小参数 λ_min 上，提出“最小参数 → 直接构造 M(q)”的路径，跳过显式 m、CoM、惯量块。 |
| **本方案一** | 1）把 **FD 行为（qdd_FD）写进目标函数**，直接对正动力学稳定性/一致性做优化；2）通过 J_M（min eig(M) 惩罚）直接控制 M(q) 的数值性质；3）全程在物理可行的 π 空间中搜索。 |
| **本方案二** | 1）用列归一化 + rank + QR 在当前 ReMatrix 上自动选取数值独立的最小参数列；2）把恢复阶段拆成不同强度的两个版本（solve_full_params_physical / recover_full_params_physical）以适配工程场景。 |

**各方法核心公式与原理**  
- **Panda**：min ‖Yπ−τ‖² + λ·penalty(π)；penalty 含 m>0、I 正定、三角不等式、质心范围等（软约束）。原理：用罚函数把物理约束写进目标，灵活但非凸。  
- **Wensing**：min ‖Yπ−τ‖² s.t. (1/2)Tr(I)I − I ≽ 0（每 link 惯量 LMI）。原理：LMI 等价于惯量正定+三角不等式，凸、全局最优。  
- **Sensors**：τ = Hλ，满秩分解 H = H₁K 得 λ_min；无显式物理约束。原理：结构保证可辨识，不约束惯量物理性。  
- **本方案一**：min J，J = w_tau‖Yπ−τ‖² + w_qdd‖qdd_FD−qdd_ref‖² + w_cad(π−π_cad)′W(π−π_cad) + w_M·J_M；nonlcon 硬约束 m∈[m_min,m_max]、|c−c_cad|≤δ_c、I 对角≥0、三角不等式、min eig(I)≥ε_I。原理：**把 FD 误差与 M 正定性写进目标**，约束与 Panda 同族但为硬约束。  
- **本方案二**：第一步 β_hat = (Y_min′Y_min)\Y_min′τ（无物理约束）；第二步 min ‖Y_full π − Y_min β_hat‖² + λR(π,π_cad) s.t. 质量/CoM/I 硬约束。原理：第一步同 Sensors 风格，第二步同 Panda 恢复风格，R 按参数类型加权。

**本单位方案与论文的区别**  
- **相对 Panda**：方案一**不**用软 penalty，用 **fmincon 硬约束**，且目标多了 **J_qdd、J_M**（直接针对 FD 与 M 病态）；方案二恢复阶段用 **Y_full π ≈ Y_min β_hat** 而非 base 映射，并可选两种恢复强度（solve_full_params_physical / recover_full_params_physical）。  
- **相对 Wensing**：本单位**无 LMI**，用**非线性不等式**描述物理约束，非凸；优点是实现简单、易与现有 ReMatrix/CAD 对接，代价是可能局部最优。  
- **相对 Sensors**：方案二第一步与 Sensors 类似（最小参数 OLS），但**不做**“λ_min 直接构造 M”；第二步显式恢复全参数并加物理约束，FD 用 π 而非 λ。

## 三、激励轨迹方案

| 方法 | 激励轨迹方案 |
| ---- | ------------ |
| **Panda 2019** | 多频正弦叠加（Fourier），以最小化 cond(Y) / 最大化 σ_min(Y) 为目标，全关节同时激励，覆盖主要频段。 |
| **Wensing 2017** | 使用随机/丰富激励，原则是回归矩阵满秩与持续激励（persistent excitation），不拘泥具体形式。 |
| **Sensors 2025** | 顺序单关节激励：一次只动一个关节，其余关节固定；实现简单、安全，但对耦合项激励不足。 |
| **本方案一** | 激励由 Excitation_Trajectory_E1 生成：Fourier + 条件数优化（与 Panda 同思路），再通过 CSV 取中间激励窗、qd 低通、qdd 差分+SG；为保证 J_qdd 有意义，需较高质量的 qdd_ref。 |
| **本方案二** | 与方案一共用同一套激励与预处理（Fourier + cond(Y_min) 优化、连续窗、滤波、可选周期平均）；对第一步最小参数辨识而言，更关注 τ 拟合，对 qdd 质量要求略低。 |

**各方法核心公式与原理**  
- **Panda**：q(t) = Σ_k [a_k sin(ω_k t) + b_k cos(ω_k t)]；优化目标最小化 cond(Y) 或最大化 σ_min(Y)。原理：Fourier 覆盖频段，全关节同时激励，条件数优化保证可辨识。  
- **Wensing**：随机/丰富激励；要求 Y 满秩、持续激励（persistent excitation）。原理：不固定轨迹形式，强调激励充分性。  
- **Sensors**：顺序单关节激励（一次只动一关节，其余固定）。原理：安全、易实现，但耦合项激励弱。  
- **本方案一**：轨迹由 Excitation_Trajectory_E1 生成（Fourier + 最小化 cond(Y_min)）；数据取窗、qd 低通、qdd 差分+SG；需较准的 qdd_ref 以支撑 J_qdd。原理：与 Panda 轨迹思想一致，数据端强调“激励段+滤波”，方案一额外依赖 qdd 质量。  
- **本方案二**：与方案一同一套轨迹与预处理（Fourier + cond(Y_min)、取窗、滤波、可选周期平均）。原理：第一步主要拟合 τ，对 qdd 要求可略低于方案一。

**本单位方案与论文的区别**  
- **相对 Panda**：激励形式与目标一致（Fourier + 条件数），本单位用 **cond(Y_min)**（QR 选列后的最小参数回归矩阵），数据使用上增加**取窗、低通、SG 求 qdd、可选周期平均**，与工程采集流程一致。  
- **相对 Wensing**：本单位**固定**为 Fourier + 条件数优化，而非随机激励；便于与 ReMatrix 和 TLBO/standalone 脚本对接。  
- **相对 Sensors**：本单位采用**多关节同时激励**（与 Panda 同），**不用**顺序单关节；目的是更好激励耦合项、提高 Y_min 条件数。

## 四、辨识方案

| 方法 | 辨识方案 |
| ---- | -------- |
| **Panda 2019** | 线性 OLS 得到 base/最小参数；再做带物理 penalty 的非线性优化，恢复全参数（软约束：质量正、惯量正定、三角不等式、质心范围等）。 |
| **Wensing 2017** | 在 τ = Yπ 上直接做凸优化 / SDP：目标拟合 τ，约束用 LMI 表达惯量物理可行，得到全局最优的物理一致全参数。 |
| **Sensors 2025** | 通过 τ = Hλ 做线性最小二乘，并用满秩分解（H = H₁K）得到 λ_min；无显式物理不等式，主要依赖结构与数据可辨识性。 |
| **本方案一** | 用 `identify_full_params_for_fd`：以全参数 π（减少到 42 维）为变量，最小化 J = w_tau·J_tau + w_qdd·J_qdd + w_cad·J_cad + w_M·J_M，配合质量/CoM/惯量的硬约束，用 `fmincon` 求解。 |
| **本方案二** | 第一步 `identify_min`：从 Y_full 通过 QR 选列得 Y_min，用 OLS/WLS 拟合 τ 得 β_hat（X_hat）；第二步 `solve_full_params_physical` / `recover_full_params_physical`：在物理约束与 CAD 正则下，从 β_hat 恢复全参数 π_phys。 |

**各方法核心公式与原理**  
- **Panda**：第一步 π_ls = (Y′Y)\Y′τ（或 base 最小参数）；第二步 min ‖Yπ−τ‖² + λ·penalty(π)，penalty 软约束。原理：两阶段，线性解再物理修正。  
- **Wensing**：min ‖Yπ−τ‖² s.t. (1/2)Tr(I)I−I≽0（每 link）。原理：一步凸优化，π 直接物理一致。  
- **Sensors**：τ = Hλ，H = H₁K 得 λ_min；β_hat = (H′H)\H′τ（或等价）。原理：仅线性 LS，无物理约束。  
- **本方案一**：变量 x = [m,mx,my,mz,Ixx,Iyy,Izz] 每 link（42 维），Ixy,Ixz,Iyz 固定为 CAD；min J，J 含 J_tau、J_qdd、J_cad、J_M；lb/ub 与 nonlcon 给出 m、CoM、I 可行域；fmincon(SQP)。原理：**单阶段**全参数辨识，目标同时含 ID 拟合、FD 拟合、CAD 先验、M 正定惩罚。  
- **本方案二**：第一步列归一化 + rank + QR 得 index_base，Y_min = Y_full(:,index_base)，β_hat = (Y_min′Y_min)\Y_min′τ（可 WLS）；第二步 min ‖Y_full π − Y_min β_hat‖² + λR(π,π_cad) s.t. 质量/CoM/I 硬约束，fmincon。原理：第一步与 Sensors 类似（最小参数 LS），第二步与 Panda 类似（约束恢复），R 按 m/h/I 分组加权。

**本单位方案与论文的区别**  
- **相对 Panda**：方案一**无**“先 base 再恢复”的两步，而是**一步**优化 π，且目标含 **J_qdd、J_M**，约束为**硬约束**；方案二有两步，但最小参数用 **QR 选列** 而非解析 base，恢复目标为 **Y_full π ≈ Y_min β_hat** + CAD 正则。  
- **相对 Wensing**：本单位**不用 SDP/LMI**，用 **fmincon + 非线性不等式**，非凸；实现简单，可与现有 ReMatrix、CAD 直接配合。  
- **相对 Sensors**：方案二第一步同 Sensors（最小参数 LS），但**必做第二步**恢复 π 并加物理约束再做 FD，不做“仅用 λ/β 做 FD”的单一路径。

## 五、正动力学方案

| 方法 | 正动力学方案 |
| ---- | ------------ |
| **Panda 2019** | 恢复的全参数喂给标准刚体动力学（如 CRBA/RNEA），构造 M(q)、h(q,qd)，再做 qdd = M⁻¹(τ−h)。 |
| **Wensing 2017** | 辨识出来的惯量本身满足 LMI 物理约束，可直接用于构造 M(q)、h(q,qd)，同样用 qdd = M⁻¹(τ−h)。 |
| **Sensors 2025** | 在空间动力学框架下，从 λ_min 构造 G = Kλ，再通过 AᵀΓᵀGΓA 得到 M(q)，从而做 FD，不显式使用 m、CoM、I 块。 |
| **本方案一** | 在 `forward_dynamics_full` 中，用 ReMatrix_E1_limb_URDF + π_fd 计算：h(q,qd) = ID(q,qd,0)，M(q) 第 j 列 = ID(q,0,e_j) − τ_g，然后用（可对称化/轻微正定修补的）M 做 qdd = M\(τ−h)。 |
| **本方案二** | **两种选择**：① **forward_dynamics_min**：直接用 X_hat、index_base，用 Y_min·β 构造 M、h，qdd = M\\(τ−h)（类 Sensors，不恢复 π）；② **forward_dynamics_full**：用恢复得到的 π_phys 构造 M、h（类 Panda）。推荐用 π_phys + forward_dynamics_full 以保证 M 物理正定。 |

**各方法核心公式与原理**  
- **Panda**：qdd = M(q)^{-1}(τ − h(q,qd))；M、h 由恢复后的全参数 π 经 CRBA/RNEA（或等价）得到。原理：标准刚体动力学，π 物理可行故 M 正定。  
- **Wensing**：同上 qdd = M^{-1}(τ−h)；π 已满足 LMI，直接构造 M、h。原理：与 Panda 一致，无单独恢复步骤。  
- **Sensors**：M(q) = f(λ_min)，由空间惯量 G = Kλ 经 AᵀΓᵀGΓA 等得到；qdd = M^{-1}(τ−h)。原理：**不显式恢复 m、CoM、I**，用 λ_min 直接构造 M。  
- **本方案一**：qdd = M\\(τ−h)；h = ID(q,qd,0)，M 第 j 列 = ID(q,0,e_j)−τ_g，ID = ReMatrix_E1_limb_URDF·π_fd。实现：**forward_dynamics_full**。原理：与 Panda/Wensing 一致（全参数 → M、h），M、h 用回归矩阵列构造而非单独 CRBA 调用。  
- **本方案二**：两种实现。① **forward_dynamics_min**：h = ID_min(q,qd,0)，M 列 = ID_min(q,0,e_j)−τ_g，ID_min = Y_min·X_hat，qdd = M\\(τ−h)（类 Sensors）。② **forward_dynamics_full(π_phys)**：同方案一（类 Panda）。原理：可选“最小参数直接 FD”或“恢复 π 后 FD”，推荐后者以保证 M 物理正定。

**本单位正动力学实现**：  
- **forward_dynamics_full(q, qd, tau, π, limb, para_order)**：全参数 π，τ = Y·π，M、h 由 ReMatrix 列构造。  
- **forward_dynamics_min(q, qd, tau, X_hat, index_base, limb, para_order)**：最小参数 β(X_hat)，τ = Y_min·β，M、h 由 Y_min 列构造；不依赖全参数，但 M 未必物理正定，易病态。

**本单位方案与论文的区别**  
- **相对 Panda/Wensing**：FD 公式一致（qdd = M^{-1}(τ−h)），区别在于 **M、h 的构造方式**：本单位用 **ReMatrix 逐列算 ID**（M 第 j 列 = ID(q,0,e_j)−τ_g）得到 M、h，与 CRBA/RNEA 在约定一致时等价；不单独调用 Toolbox 的 CRBA/RNEA。  
- **相对 Sensors**：Sensors 用 λ_min 经**空间惯量 G = Kλ** 构造 M；本单位若用 **forward_dynamics_min** 则形式类似（β 直接参与构造 M、h），但 M 由 **Y_min 列**按同一 ID 列公式构造，**不**用空间惯量分解，且 M 易病态，故推荐用 **forward_dynamics_full(π_phys)**，即仍走“全参数 → M、h”的 Panda 路线。

## 六、测试与结果

**验收参考与 CAD 的角色（实机/实测数据）**  
- **最终参考目标**：实测或预处理得到的 **关节角 q、速度 qd、加速度 qdd**（以及由采集得到的力矩 τ）。辨识与验证的优劣以**与这些轨迹/力矩的一致程度**为准。  
- **CAD（π_cad）**：仅作**先验或参考**，用于正则化（如 J_cad、R(π,π_cad)）、约束边界（如 m 区间、CoM 范围）和初值；**不作为**验收真值。即：不要求 π 逼近 π_cad，而是要求 ID/FD 预测逼近**实测 q、qd、qdd、τ**。  
- **ID 验收**：以**实测或预处理力矩 τ_meas** 为参考，比较 Y·π 或 Y_min·β 的预测误差。  
- **FD 验收**：以**实测或估计的 qdd**（及轨迹 q、qd）为参考，比较 qdd_FD 与 qdd 的误差；q、qd 一致性也可作为辅助检查。

| 方法 | 测试与结果（概念层） |
| ---- | -------------------- |
| **Panda 2019** | 在机械臂平台上验证 τ 拟合与 qdd 预测：证明在加入物理约束并做参数恢复后，既保持 τ 拟合精度，又能用恢复参数做稳定的 FD。 |
| **Wensing 2017** | 通过仿真与实验对比，有无 LMI 约束时的惯量质量与 FD 行为，展示 LMI 能有效压制非物理惯量，提升在噪声条件下的稳定性。 |
| **Sensors 2025** | 在仿真/实验中展示用 λ_min 构造的 M(q) 能在一定范围内给出合理 qdd，且时序上更紧凑；但对数值稳定性和噪声较敏感。 |
| **本方案一** | 仿真/实机上用激励 CSV：先用 `identify_full_params_for_fd` 得 π_fd，再在 `run_full_dynamics_validation` 等脚本中对比 qdd_FD 与实测 qdd；预期现象是 FD 更稳定、M(q) 特征值/条件数更健康。 |
| **本方案二** | 先 `run_min_param_id_from_csv` 得 X_hat，再恢复 π_phys 并进 `forward_dynamics_full`；对比“只用 X_hat 直接 FD”与“恢复后 FD”的差异，可见 π_phys 提升了 M(q) 正定性与 FD 的可用性；若仍不足，可增强恢复约束或切换方案一继续优化。 |

**各方法核心公式与原理**  
- **Panda**：验证 τ 拟合误差、用恢复参数做 FD 得到的 qdd 与实测/仿真 qdd 的误差；说明恢复后参数既拟合 ID 又支持稳定 FD。原理：两阶段后整体验证。  
- **Wensing**：对比有无 LMI 约束下惯量质量、FD 稳定性与噪声鲁棒性。原理：凸约束带来可重复性与稳定性。  
- **Sensors**：验证用 λ_min 构造的 M(q) 做 FD 的 qdd 误差；讨论数值稳定性与噪声敏感度。原理：最小参数直接 FD 的可行性边界。  
- **本方案一**：指标包括 J_tau/J_qdd/J_cad/J_M、τ 与**实测/预处理 τ** 的 RMSE、qdd_FD 与**实测/估计 qdd** 的 RMSE、min(eig(M))、cond(M)；脚本如 run_full_dynamics_validation。原理：辨识目标与验证指标一致，**真值为实测 q、qd、qdd（及 τ）**，CAD 仅参与正则与约束。  
- **本方案二**：第一步看 τ 与**实测 τ** 的拟合 RMSE、cond(Y_min)；第二步看 ‖Y_full π_phys − Y_min β_hat‖、恢复后 min(eig(I))；FD 对比“forward_dynamics_min(X_hat)”与“forward_dynamics_full(π_phys)”以及**实测 qdd**。原理：分步验证，**最终以轨迹 q、qd、qdd 为参考**，对比“仅最小参数 FD”与“恢复后 FD”的差异。

**本单位方案与论文的区别**  
- **相对 Panda**：本单位方案一在验证时**直接看 J_qdd、min(eig(M))**，与辨识目标一致；方案二额外做 **forward_dynamics_min vs forward_dynamics_full(π_phys)** 对比。**实机下验收真值为实测 q、qd、qdd（及 τ）**，CAD 仅作先验，不作为“逼近目标”。  
- **相对 Wensing**：本单位无 LMI，验证时更依赖 **min(eig(M))**、cond(M)、**与实测 qdd 的误差**等数值指标；CAD 一致性检查（π_cad/β_cad 下 FD 与 Toolbox 一致）仅用于**自检实现正确性**，非实机验收主指标。  
- **相对 Sensors**：本单位若用 forward_dynamics_min 可做“最小参数直接 FD”的验证，但会同时验证 **forward_dynamics_full(π_phys)**；**实机验收以实测加速度/速度/关节角为最终参考**，不以 CAD 或 URDF 为真值。

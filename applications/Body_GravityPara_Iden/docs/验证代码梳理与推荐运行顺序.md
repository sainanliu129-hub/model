# Body_GravityPara_Iden 验证流程说明（排错作战手册）

**核心原则：先模型链路，后辨识，再 FD，最后三模型。阶段 1 必须先全过，再碰阶段 2 之后的东西。**

若 `test_ReMatrix_E1_limb_URDF` 或 `test_min_param_URDF` 没过，后面 `run_min_param_id_from_csv`、`run_check_full_param_consistency`、`run_full_dynamics_validation`（或 `run_debug_pi_phys_id_fd_plot`）都只是在错误底座上堆实验。

---

## 一、关键停止条件（失败即停）

| 条件 | 含义 | 失败后动作 |
|------|------|------------|
| **停止条件 1** | `test_ReMatrix_E1_limb_URDF` 不通过 | **停止后续全部流程**，先修 ReMatrix / URDF / get_limb_theta_from_URDF。 |
| **停止条件 2** | **CAD 全参链路一致性**不通过（`run_check_full_param_consistency_cad_only`） | 禁止跑 β→π、FD_full、三模型对比；先跑 `run_diagnose_pi_Y_columns`（或 `run_check_pi_order_vs_urdf` / `run_check_regressor_column_norms`）修链路。 |
| **停止条件 3** | `test_min_param_URDF` 不通过 | 禁止用真实数据做辨识分析，先修 `identify_min`。 |

**阶段 C（FD 验证）** 只有在 **A1 与 A3 均通过** 后才进入，否则图示只会制造干扰。

---

## 二、推荐验证顺序（最终建议版）

### 阶段 A：基础模型一致性

| 步 | 脚本 | 目标 | 前置 |
|----|------|------|------|
| **A1** | `test_ReMatrix_E1_limb_URDF`（`trajectory_source='random'`） | τ_ref ≈ Y·θ_urdf | 无 |
| **A2** | `test_min_param_URDF` | identify_min 自洽；X_hat ≈ X_theta（理论最小参） | 无 |
| **A3** | `run_check_full_param_consistency_cad_only` | Y_full·π_cad ≈ τ_urdf；FD_full(π_cad) ≈ FD_urdf；单参扰动有效 | **不依赖 mat**，可用 random 或 csv |
| **A4** | 若 **A3 任一项未过**，立刻跑：`run_diagnose_pi_Y_columns` | 查 π 与 Y 列对应、列范数、有限差分、静态重力 | 无 mat 时脚本内用随机轨迹 |

**只要下面任一项不通过，就立刻跑 `run_diagnose_pi_Y_columns`（不必等辨识）：**

- τ_full(π_cad) vs τ_urdf 不通过  
- FD(π_cad) vs FD_urdf 不通过  
- 单参数扰动无响应  
- Y_full·π_cad 和直觉不符  

---

### 阶段 B：真实辨识

| 步 | 脚本 | 目标 | 前置 |
|----|------|------|------|

| **B2** | `run_torque_comparison` | 先只看 ID（τ 对比），不看 FD | 可选 mat |

### 阶段 C：FD 验证

| 项目 | **run_check_full_param_consistency_cad_only** | **run_check_full_param_consistency** |

## 六、ReMatrix 与 subtree base（link1 前 9 列为 0 时）

- **原因**：若 `get_e1_limb_robot` 返回的 subtree 满足 **BaseName == Bodies{1}.Name**（例如从 `leg_l1_link` 截取的子树以自身为 base），则 `getTransform(..., Bodies{1}, baseName)` 恒为单位阵，第一体被当作固定基座，Y 的前 9 列（m, mx, my, mz, Ixx, Ixy, Ixz, Iyy, Iyz）全为 0，仅 Izz 非零；静态重力测试改 link1 质量 τ 不变。
- **修正**：`ReMatrix_E1_limb_URDF` 在检测到 **BaseName == Bodies{1}.Name** 时，自动改用整机 `get_e1_full_robot` 并以第一体的**父体**为 base 计算 T0，使 T0{1} 非单位阵，从而 link1 的 10 列均能参与回归。依赖 `get_e1_full_robot`、`get_e1_full_robot_limb_indices`（需 `robot_model` 在路径）。
- **若仍为 0**：若你的 subtree 的 BaseName 已是父体（如 `base_link`），则 link1 前 9 列仍可能因**几何/轨迹**（例如第一关节轴竖直、重力沿轴、静态下无激励）而近似为 0，此时属未激励而非列序错误。

## 七、最小参数 β 与全参恢复 π_rec 的使用建议

- **模型主线**：在辨识与正动力学中，以最小参数模型为主，即使用 `Y_min·β` 和 `forward_dynamics_min`。`test_min_param_URDF` 与 `test_recover_full_params_from_beta` 已验证：在纯 URDF 下，`Y_min·β_urdf ≈ Y_full·θ_urdf`，且经约束恢复得到的 `π_rec` 满足 `Y_full·π_rec ≈ Y_full·θ_urdf`。
- **全参恢复用途**：`recover_full_params_from_beta` 采用约束最小二乘 `min_π ||Kπ-β_hat||^2 + λ||π-π_cad||^2`，得到的 `π_rec` 可用于“贴近 CAD 的等价 full 参数”，适合用于 **更新 URDF 惯量、用 Toolbox 的 forwardDynamics/inverseDynamics 做对比**。
- **注意事项**：旧的基于导出 S 矩阵的恢复方式（直接用 `S*β`）已在纯 URDF 校核中证明会导致 `Y_full·π_rec` 与 `Y_min·β` 差异很大，**不再推荐使用**。若需要 full 参，只应通过 `recover_full_params_from_beta` 一条链路获得。
- **不保证物理唯一性**：最小参数 β 可以唯一确定动力学，但恢复得到的 full 参数 π_rec 在物理上通常不唯一，仅保证在 torque 空间与 β 等价；不应期望从 π_rec 反推出真实的每一个 link 惯量/质心，只能把它视为“在最小参约束下、尽量贴近 CAD 的一组等价惯性参数”。

---

## 八、forward_dynamics_full 与 forwardDynamics 原理一致性

- **同一组动力学方程**：两者求解的都是 τ = M(q)·q̈ + h(q,q̇)，其中 h = C·q̇ + g。在 **Y·π 与 inverseDynamics(robot_limb) 一致**、且 **get_limb_theta_from_URDF(robot_limb) = π** 的前提下，由 ReMatrix 构造的 M、h 与 Toolbox 内部使用的 M、h 在原理上相同，故 **理论上** 两种实现应给出相同 q̈。
- **不能先验保证**：一致性依赖 ReMatrix 与 Toolbox 在运动学（T0、关节轴）、重力方向、惯量/质心顺序上完全一致；任何一处约定不同都会导致差异。因此 **只能通过数值自检保证**（如 `run_check_full_param_consistency_cad_only` 自检 2、`run_full_dynamics_validation` 中的 FD(π_cad) vs FD_urdf）。
- **用“更新树”做交叉验证**：`set_limb_theta_to_URDF(robot_limb, π)` 可根据全参 π 更新 rigidBodyTree 的 Mass/CenterOfMass/Inertia（不修改原树）。用更新后的树调用 `forwardDynamics(robot_updated, q, qd, tau)` 应与 `forward_dynamics_full(q, qd, tau, π, limb, para_order)` 结果一致（在 set/get_limb_theta 与 ReMatrix 一致时）。在 `run_full_dynamics_validation` 中设 `fd_use_urdf_updated = true` 可改为用“更新树 + forwardDynamics”计算 π_rec/π_phys 的 FD，用于与 forward_dynamics_full 交叉验证。

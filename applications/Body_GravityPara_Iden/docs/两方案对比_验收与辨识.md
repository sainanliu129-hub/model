# 两方案对比：test_ReMatrix_E1_limb_URDF（验收） vs run_min_param_id_from_csv（辨识）

## 辨识只需两样东西（不用 θ）

- **Y_min**：由轨迹 (q, qd, qdd) → ReMatrix 得 Y → 列归一化 + rank + QR 得 index_base → Y_min = Y(:, index_base)。**全程不涉及 θ**。
- **τ_meas**：实测力矩，按点堆叠成列向量 b。

解 **Y_min * X_hat ≈ τ_meas** 即 **X_hat = Y_min \ τ_meas**，即得最小动力学参数。θ 只在验收脚本里出现（Y*θ、β=H*θ 等），辨识中不需要。

---

## 一、两个方案分别是什么、做什么用

| 项目 | test_ReMatrix_E1_limb_URDF | run_min_param_id_from_csv + identify_min |
|------|----------------------------|------------------------------------------|
| **角色** | 验收脚本：检验 ReMatrix 与逆动力学是否一致，并做“同轨迹”下的多种对比 | 辨识流程：从 CSV 实测数据估计最小惯性参数，并画测量 vs 预测 |
| **主要目的** | 1）验证 Y*θ 与 inverseDynamics 的 τ_ref 是否一致（全参）<br>2）验证 Y_min*β 与 τ_ref 的关系（最小参数，β=H*θ 或 X_hat=Y_min\τ_ref）<br>3）CSV 模式下额外画 τ_meas、τ_ref、Y_min*X_hat_meas 等对比 | 从 τ_meas 辨识出 X_hat，得到 τ_pred=Y_min*X_hat，并画 τ_meas vs τ_pred、保存 X_hat 供 FD 等使用 |
| **“参考”力矩** | **τ_ref = inverseDynamics(q,qd,qdd)**（模型逆动力学）为主；CSV 时另有 **τ_meas**（实测）做对照 | **τ_meas**（CSV 实测力矩） |
| **是否依赖 URDF 的 θ** | 是：用 get_limb_theta_from_URDF 得到 θ，算 Y*θ、H*θ 等 | 否：不读 θ，只从 τ_meas 求 X_hat |

---

## 二、相同点（保持一致的部分）

### 1. 轨迹与数据来源（CSV 模式下）

- **同一 CSV、同一时间窗、同一套预处理** 时，两方案应使用**完全相同的 (q, qd, qdd, τ)**：
  - 默认窗：`t_start_s=2`, `t_end_s=4`
  - 预处理：`continuous_window_id_data(..., opts)`，其中 `qd_lowpass_fc_Hz=50`, `qdd_smooth_half=15`
- 即：**q_bar / qd_bar / qdd_bar / tau_bar**（辨识）与 **q_all / qd_all / qdd_all / tau_meas_all**（验收 CSV 模式）来自同一函数、同一 opts，点数 M = nTest 一致。

### 2. 观测矩阵 Y 的构造

- 都使用 **ReMatrix_E1_limb_URDF(limb, q, qd, qdd, 1, para_order)** 得到**全参**观测矩阵 Y（行数 = 点数×6，列数 = 60）。
- **行顺序**：与 ReMatrix 一致，为**按点堆叠**  
  `[点1 的 6 行; 点2 的 6 行; …]`（point-major）。

### 3. 最小参数列的选取（index_base 与 Y_min）

两处**完全一致**的步骤：

- 列归一化：`col_norm = sqrt(sum(Y.^2,1))`，`W = Y ./ (ones(...,1)*col_norm)`
- 定秩：`r = rank(W)`
- 列主元 QR：`[~,~,piv] = qr(W,'vector')`，`index_base = sort(piv(1:r))`
- 最小观测矩阵：**Y_min = Y(:, index_base)**

因此，在**同一条轨迹**（同一 Y）下，两方案得到的 **index_base** 和 **Y_min** 相同。

### 4. τ 的堆叠顺序（与 Y 行对应）

- **Y 的行**：point-major，即 `[点1 关节1…6; 点2 关节1…6; …]`。
- **τ 的列向量** 必须与 Y 同一顺序：
  - 正确：**b = reshape(tau_bar', [], 1)**（按点堆叠）。
  - 错误：`tau_bar(:)` 为按列（关节）堆叠，与 Y 行不对应。

验收脚本里 τ_ref_stack、tau_meas_stack 均为按点堆叠；identify_min 中 b 已改为 `reshape(tau_bar', [], 1)`，与验收一致。

### 5. 普通最小二乘求“最小参数”时

- 验收（CSV）：**X_hat_meas = Y_min \ tau_meas_stack**（无权重）。
- 辨识（与验收对齐时）：**X_hat = Y_min \ b**，且 **use_wls = false**，即不加权。
- 此时**同一 (q,qd,qdd,τ_meas)** 下，**X_hat 与 X_hat_meas 应相同**，**τ_pred = Y_min*X_hat** 与验收里的 **Y_min*X_hat_meas** 一致。

---

## 三、不同点

### 1. 参考力矩与“最小参数”的求法

| 对比项 | test_ReMatrix_E1_limb_URDF | run_min_param_id_from_csv / identify_min |
|--------|----------------------------|------------------------------------------|
| **参考力矩** | τ_ref = inverseDynamics（模型）；CSV 时另有 τ_meas | τ_meas（实测） |
| **用 τ_ref 时** | X_hat = Y_min \ τ_ref_stack，得到“拟合 τ_ref”的最小参数；β = H*θ = pinv(Y_min)*Y*θ 为“由全参 θ 映射到最小参数” | 不涉及 τ_ref、θ |
| **用 τ_meas 时** | X_hat_meas = Y_min \ tau_meas_stack，仅作对比与验收 | X_hat = Y_min \ b（b=τ_meas 按点堆叠），即辨识结果；可选 WLS（use_wls=true 时） |
| **H = pinv(Y_min)*Y** | 用于**有 θ 时**：β = H*θ，满足 Y_min*β ≈ Y*θ，做全参→最小参数映射与 cover 验证 | **不用**：辨识时没有 θ，只有 τ_meas，故用 Y_min\τ_meas 求 X_hat，不用 H*θ |

总结：**H*θ** 是“已知全参 θ 时，映射到最小参数 β”；**辨识**是“没有 θ，用实测 τ_meas 直接求最小参数 X_hat”。

### 2. 加权最小二乘（WLS）

- **identify_min** 可选 **use_wls**：
  - **use_wls = true**：用 avg_data.tau_std 做权重，Aw\bw；与 test_ReMatrix 的**普通** LS 不同，解会不同。
  - **use_wls = false**：普通 LS，与验收脚本中 **Y_min \ tau_meas_stack** 一致。
- **test_ReMatrix** 中不做 WLS，只用 Y_min \ τ 的普通 LS。
- 要与验收“同方案”时，辨识应设 **use_wls = false**（run_min_param_id_from_csv 已显式传入）。

### 3. 输出与用途

- **test_ReMatrix**：打印/画图 τ_ref vs Y*θ、τ_ref vs Y_min*X_hat、τ_meas vs Y_min*X_hat_meas（CSV）；写 base_parameter_formulas.txt；**不保存 X_hat**，只做验收。
- **run_min_param_id_from_csv**：画 τ_meas vs τ_pred，保存 **X_hat、index_base** 到 min_param_id_result.mat，供 forward_dynamics_min 等后续使用。

### 4. 轨迹模式

- **test_ReMatrix**：可选 `trajectory_source = 'csv' | 'excitation' | 'random'`；CSV 时用固定 csv_file_validation 与 csv_continuous_opts。
- **run_min_param_id_from_csv**：可选 `mode = 'continuous' | 'cycle_average'`；continuous 时用 opts.continuous_opts（默认已与 test_ReMatrix 的 csv_continuous_opts 对齐：t_start_s=2, t_end_s=4, qdd_smooth_half=15 等）。

---

## 四、公式与流程对照（简要）

- **全参**  
  τ = Y*θ（θ 为 60 维，来自 URDF）。  
  验收：检查 |τ_ref − Y*θ|。

- **最小参数（有 θ 时）**  
  β = H*θ，H = pinv(Y_min)*Y，满足 Y_min*β ≈ Y*θ。  
  验收：检查 Y*θ 与 Y_min*β 的 cover、以及 τ_ref 与 Y_min*X_hat（X_hat=Y_min\τ_ref）的误差。

- **最小参数（无 θ，用实测）**  
  X_hat = Y_min \ τ_meas（按点堆叠），τ_pred = Y_min*X_hat。  
  辨识与验收在**同一轨迹、同一 Y_min、普通 LS、同一 τ 堆叠顺序**下，X_hat 与 τ_pred 一致。

---

## 五、如何保证“同轨迹、同方案”下结果一致

1. **同一 CSV、同一时间窗**：两脚本用相同的 t_start_s、t_end_s（如 2、4）。
2. **同一预处理**：continuous_window_id_data 使用相同 opts（qd_lowpass_fc_Hz=50, qdd_smooth_half=15）。
3. **同一堆叠顺序**：b = reshape(tau_bar', [], 1)（按点），不用 tau_bar(:)。
4. **辨识用普通 LS**：identify_min(..., struct('use_wls', false))。
5. 若仍不一致，检查是否误用了不同 CSV、不同窗或不同 opts（例如时间跨度 5 s 与 2 s 会对应不同轨迹段）。

---

## 六、何时用哪个

- **test_ReMatrix_E1_limb_URDF**：验证 ReMatrix 与逆动力学、最小参数公式、以及“同轨迹下 τ_meas 与 Y_min*X_hat_meas”是否一致；不改动辨识结果文件。
- **run_min_param_id_from_csv + identify_min**：从实测 CSV 辨识最小参数、得到 X_hat 与 τ_pred，并保存供 FD 等使用；要与验收一致时，使用相同窗与 opts，并设 use_wls=false。

# 动力学最小参数辨识：从采集到验证 — 流程梳理与测试报告

本文档梳理 **Body_GravityPara_Iden** 从数据采集、预处理、辨识、到验收与正动力学验证的完整流程，并给出验证标准与测试报告模板。

---

## 一、整体流程概览

```
[1] 激励轨迹生成/采集  →  [2] CSV 数据  →  [3] 预处理(窗/滤波)  →  [4] 最小参数辨识  →  [5] 验收(ReMatrix)  →  [6] FD 验证
```

| 阶段 | 主要脚本/函数 | 输出 |
|------|----------------|------|
| 1 激励与采集 | `excitation_trajectory/` 生成轨迹；实机/仿真按轨迹跑并录 CSV | 含 time, q, qd, τ 的 CSV |
| 2 读入 | `read_leg_joint_csv(csv_file)` | data.time, pos_leg_l, vel_leg_l, torque_leg_l 等 |
| 3 预处理 | `continuous_window_id_data` 或 `cycle_average` | avg_data: q_bar, qd_bar, qdd_bar, tau_bar |
| 4 辨识 | `identify_min(avg_data, 'left_leg', opts)` | X_hat, index_base, metrics（τ_pred, RMSE 等） |
| 5 验收 | `test_ReMatrix_E1_limb_URDF` | 全参/最小参数与 τ_ref、τ_meas 一致性 |
| 6 FD 验证 | `run_full_dynamics_validation`（或 `run_debug_pi_phys_id_fd_plot` 调 π_fd） | q/qd/qdd 误差统计与曲线 |

---

## 二、各阶段说明与操作步骤

### 2.1 数据采集

**要求：**

- **CSV 格式**：与 `read_leg_joint_csv` 兼容（见 `utility_function/read_leg_joint_csv.m`）。
  - **老格式**：表头含 `time, cmd_leg_l1_joint, pos_leg_l1_joint, vel_leg_l1_joint, torque_leg_l1_joint, ...`（l2..l6、右腿 r1..r6 同理）。
  - **新格式**：表头含 `time, q_1_data, q_2_data, ..., qd_1_data, ..., tau_1_data, ...`。
- **内容**：至少包含 **时间、左腿 6 关节位置 q、速度 qd、力矩 τ**。辨识用单腿（默认左腿 `pos_leg_l`, `vel_leg_l`, `torque_leg_l`）。
- **单位**：角度 rad，角速度 rad/s，力矩 N·m；列序与 E1 腿关节顺序一致。

**激励轨迹建议：**

- 使用 `excitation_trajectory/` 下生成的激励（如 TLBO 优化、傅里叶级数等），多周期（如 5 周期）并含前后过渡段。
- 采集时尽量只动一条腿（左腿或右腿），另一腿保持安全位，避免碰撞与奇异。

**存放位置：**

- 推荐：`applications/Body_GravityPara_Iden/data/excitation/` 或 `data/跑步/`。
- 若不传参，`run_min_param_id_from_csv` 会优先在上述目录查找 `*.csv`。

---

### 2.2 预处理（辨识用数据）

**两种模式（由 `run_min_param_id_from_csv(..., opts)` 的 `opts.mode` 决定）：**

| 模式 | 说明 | 适用 |
|------|------|------|
| **continuous**（默认） | 取时间窗 [t_start_s, t_end_s] → qd 低通 → qdd 由 qd 差分 + SG 平滑 | 单段激励、快速辨识 |
| **cycle_average** | 周期对齐 + 相位平均，需周期数 K、周期 T，可选 ref_t/ref_q | 多周期重复激励、降噪 |

**与验收一致时的连续窗默认（推荐）：**

- `t_start_s = 2`, `t_end_s = 4`（与 `test_ReMatrix_E1_limb_URDF` 的 `csv_continuous_opts` 一致）。
- `qd_lowpass_fc_Hz = 50`，`qdd_smooth_half = 15`（qdd 必须平滑，否则 τ_pred 噪声大、RMSE 偏高）。

**调用示例：**

```matlab
continuous_opts = struct('t_start_s', 2, 't_end_s', 4, ...
    'qd_lowpass_fc_Hz', 50, 'qdd_smooth_half', 15);
avg_data = continuous_window_id_data(t, q, qd, tau, continuous_opts);
% avg_data 含: q_bar, qd_bar, qdd_bar, tau_bar, t_equiv, (可选 tau_std)
```

---

### 2.3 最小参数辨识

**核心：** 解 **Y_min·X_hat ≈ τ_meas**（τ 按点堆叠为列向量 b），即 **X_hat = Y_min \ b**。

- **Y_min**：由 (q_bar, qd_bar, qdd_bar) 经 `ReMatrix_E1_limb_URDF` 得到全参 Y，再列归一化 → rank → QR 列主元得 `index_base`，Y_min = Y(:, index_base)。**不涉及 URDF 的 θ**。
- **b**：`reshape(tau_bar', [], 1)`（按点堆叠，与 Y 行序一致）。

**与验收对齐：** 使用 **普通最小二乘**（`use_wls = false`），这样同一轨迹下 `identify_min` 的 X_hat 与验收脚本中 `X_hat_meas = Y_min \ tau_meas_stack` 一致。

**一键运行：**

```matlab
% example_run_min_param_id.m 中默认：连续窗 [2,4]s，普通 LS
run_min_param_id_from_csv(csv_file, struct('mode', 'continuous', 'continuous_opts', continuous_opts));
```

**输出：**

- 辨识图：τ_meas vs τ_pred（各关节 RMSE 打印）。
- `min_param_id_result.mat`：X_hat, index_base, avg_data, metrics（供 FD 验证使用）。

---

### 2.4 验收（ReMatrix 与逆动力学一致性）

**脚本：** `test_ReMatrix_E1_limb_URDF.m`

**作用：**

1. **全参验收**：τ_ref = inverseDynamics(q, qd, qdd)，τ_Y = Y*θ（θ 来自 URDF），检查 max\|τ_ref − Y*θ\| ≤ tol_norm（默认 0.5 N·m）。
2. **最小参数验收**：Y_min*β ≈ Y*θ（β = H*θ），以及 τ_ref 与 Y_min*X_hat（X_hat = Y_min\τ_ref）的误差。
3. **CSV 模式**：同轨迹下再画 τ_meas、τ_ref、Y_min*X_hat_meas（X_hat_meas = Y_min\τ_meas），用于和辨识脚本对比。

**轨迹与辨识一致：**

- `trajectory_source = 'csv'`，且 `csv_file_validation`、`csv_continuous_opts` 与 `example_run_min_param_id` 使用**同一 CSV、同一时间窗、同一 opts**，这样验收中的「实际力矩 vs 实际辨识最小参数预测」与 `run_min_param_id_from_csv` 的 τ_pred 一致。

**通过标准（脚本内）：**

- 全参：max\|τ_ref − Y*θ\|_∞ ≤ tol_norm。
- 最小参数：max\|Y*θ − Y_min*X_urdf\|_∞、max\|τ_ref − Y_min*X_hat\|_∞ ≤ tol_norm。

---

### 2.5 正动力学（FD）验证

**脚本：** `run_full_dynamics_validation.m`（或 `run_debug_pi_phys_id_fd_plot.m` 调 π_fd）

**前置：** 已运行 `example_run_min_param_id` 生成 `min_param_id_result.mat`。

**两种验证方式：**

| 模式 | use_test_trajectory | 验证轨迹 | 用途 |
|------|---------------------|----------|------|
| 同轨迹 | false（默认） | mat 内保存的辨识用轨迹 | 检查 FD 实现与辨识自洽 |
| 换轨迹 | true | 另一 CSV + test_window_opts（如 [4,6]s） | 检查泛化、不同时段 |

**流程简述：**

1. 加载 X_hat, index_base；加载验证轨迹 (q_meas, qd_meas, τ_meas, qdd_meas)。
2. 逐点计算 qdd_fd = forward_dynamics_min(q, qd, τ, X_hat, index_base, 'left_leg', 1)。
3. 用梯形积分从 qdd_fd 得到 qd_fd、q_fd。
4. 统计误差：RMSE、max\|err\|、MAE（对 q、qd、qdd 分别）；并绘图。

**验证标准（见下节）：** 主要看 qdd 的 RMSE/max 误差是否在可接受范围，以及 q/qd 积分漂移是否合理。

## 三、验证标准与判定

### 3.1 验收脚本（test_ReMatrix）

| 项目 | 标准 | 判定 |
|------|------|------|
| 全参 \|τ_ref − Y*θ\| | max\|·\|_∞ ≤ 0.5 N·m（tol_norm） | 通过/未通过 |
| 最小参数 Y*θ vs Y_min*β | cover 误差为数值量级 | 通过/未通过 |
| 最小参数 τ_ref vs Y_min*X_hat | max\|·\|_∞ ≤ 0.5 N·m | 通过/未通过 |
| CSV：τ_meas vs Y_min*X_hat_meas | 无硬性阈值；若相对残差很大需查 qdd 平滑与数据 | 参考 |

### 3.2 辨识结果（identify_min / run_min_param_id_from_csv）

| 指标 | 含义 | 参考（可依实际调整） |
|------|------|----------------------|
| 各关节 RMSE (N·m) | τ_pred 与 τ_meas 的均方根误差 | 越小越好；与量级和噪声有关 |
| 各关节 max\|err\| (N·m) | 最大绝对误差 | 无系统偏置时不应过大 |
| 相对残差 \|Y_min*X_hat−b\|/\|b\| | 拟合程度 | <0.5 较理想；>0.5 提示模型/数据/平滑问题 |

### 3.3 FD 验证（run_full_dynamics_validation / run_debug_pi_phys_id_fd_plot）

| 指标 | 含义 | 参考（可依实际调整） |
|------|------|----------------------|
| qdd RMSE (rad/s²) | FD 计算的 qdd 与窗内差分+平滑的 qdd_meas 的差异 | 与 qdd 量级和窗长有关，可设阈值（如 <10） |
| qdd max\|err\| | 瞬时最大误差 | 排除尖峰后评估 |
| q/qd RMSE、max\|err\| | 由 qdd 积分带来的漂移 | 同轨迹短窗应较小；换轨迹可放宽 |

**建议：** 先在同轨迹下看 qdd 误差是否可接受，再换轨迹看泛化；若同轨迹 qdd 误差就很大，优先检查辨识与 FD 实现、时间窗与平滑参数。

## 五、脚本调用顺序小结

1. **仅做验收（不依赖新采集数据）：**  
   运行 `test_ReMatrix_E1_limb_URDF`（可选 trajectory_source = 'csv' 并用现有 CSV）。

2. **从采集到辨识再到 FD 验证：**
   - 将 CSV 放到 `data/excitation/`（或 `data/跑步/`）。
   - 运行 `example_run_min_param_id`（或自行调用 `run_min_param_id_from_csv(csv_file, opts)`）→ 得到 `min_param_id_result.mat`。
   - 运行 `run_full_dynamics_validation`（在脚本里改 CSV/时间窗即可做“同轨迹/换轨迹”FD 对比），或用 `run_debug_pi_phys_id_fd_plot` 单独调 π_fd 并画 ID/FD。

3. **保证与验收一致：**  
   使用相同 CSV、相同连续窗（如 [2,4]s）、相同 `continuous_opts`（qd 50Hz、qdd_smooth_half=15），且辨识时 `use_wls=false`。详见 `两方案对比_验收与辨识.md`。

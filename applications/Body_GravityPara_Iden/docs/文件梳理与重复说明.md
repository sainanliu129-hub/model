## 三、功能重叠（非完全重复，可选合并或保留多入口）

### 1. 主流程 vs 子集入口

| 脚本 | 作用 | 关系 |
|------|------|------|
| **run_light_dynamics_validation** | **轻量版全流程（推荐）**：可选“轨迹A辨识/轨迹B对比”，按 cfg 选择模型/预处理/时间窗，并可导出 Excel | 建议替代日常频繁调参场景 |
| **run_full_dynamics_validation** | 全流程：Step2 辨识 → Step3 π_rec → Step3b π_fd + π_phys → Step4 ID 对比 → Step5 FD 对比；写 fd_cache.mat | 重流程、最完整，适合最终交叉验证 |

**建议**：日常调参/换轨迹优先用 **run_light_dynamics_validation**；需要逐步排错/交叉验证时再跑 **run_full_dynamics_validation**。

### 2. 力矩对比：run_torque_comparison vs run_min_param_id_from_csv

- **run_torque_comparison**：多版本力矩对比（刚体 / 刚体+摩擦+转子 / 辨识），可独立选 CSV、预处理选项，可不依赖 mat。
- **run_min_param_id_from_csv**：内含「3b. 多版本力矩对比」一节（当 use_preprocess_id + continuous 时），与 run_torque_comparison 思路一致。

**建议**：文档已说明可抽成同一预处理+力矩计算封装，两处调用。当前保留两脚本即可，后续若改预处理约定，只需改一处封装。

### 3. 批量汇总 vs 全流程

| 脚本 | 作用 | 关系 |
|------|------|------|
| **run_batch_id_summary** | 用 mat 生成多类型参数（CAD/min/β_cad/π_rec/π_phys/π_fd）的 ID 误差表、参数摘要表、拟合图；写 id_param_sets.mat | 不跑辨识、不跑 Step3b，只读 mat + 可选 fd_cache 中的 π_fd |
| **run_batch_fd_summary** | 用 id_param_sets.mat（及可选 fd_cache 轨迹）算各类型 FD，生成加速度误差表与图 | 依赖 run_batch_id_summary 产出的 id_param_sets.mat |
| **run_full_dynamics_validation** | 全流程含辨识、π_rec、π_fd、π_phys、ID/FD 对比，并写 fd_cache.mat | 与 batch 不重复：batch 只做“表+图”，不跑优化 |

**建议**：保留。需要表格/图汇总时先跑 **run_full_dynamics_validation**（或至少 run_min_param_id_from_csv），再跑 **run_batch_id_summary** → **run_batch_fd_summary**。

### 4. 示例与底层对比脚本

| 脚本 | 说明 |
|------|------|
| **example_compare_torque_sim_vs_id.m** | 仅配置 + 调用 compare_torque_sim_vs_id，不重复 |
| **example_compare_single_joint_torque.m** | 仅配置 + 调用单关节对比，不重复 |
| **compare_torque_sim_vs_id.m** | 底层：实机/仿真力矩 vs 逆动力学，含对齐、摩擦、可选 FD | 与 run_torque_comparison 不同（后者用 min 参数辨识结果） |

无重复，保留即可。

## 五、test/ 下脚本按用途分类

### 主流程 / 入口

- run_min_param_id_from_csv.m — 从 CSV 做辨识，写 min_param_id_result.mat  
- run_full_dynamics_validation.m — 全流程（辨识 + π_rec + π_fd + π_phys + ID/FD），写 fd_cache.mat  
- run_light_dynamics_validation.m — **轻量版全流程（推荐）**：选轨迹A/轨迹B、选模型、可导出 Excel  
- run_debug_pi_phys_id_fd_plot.m — **π_fd 调参入口**：调用 `identify_full_params_for_fd(..., opts_fd)` 得 π_fd，并画 ID/FD；当前默认 **不跑 π_phys**  
- run_identify_pi_fd_only.m — **统一辨识入口（函数）**：按 cfg.models 选择跑 pi_rec/pi_fd/pi_phys，并落盘到 build/id_param_sets.mat  
- run_plot_id_fd_only.m — **统一验证/绘图入口（函数）**：只读 mat，按 cfg.models 做 ID/FD 对比（不跑优化）  
- run_iden_full_workflow.m — 配置 + 调用 run_min_param_id_from_csv（预处理/摩擦可选）  
- run_batch_id_summary.m — 多类型参数 ID 误差/参数表/图，写 id_param_sets.mat  
- run_batch_fd_summary.m — 用 id_param_sets 做 FD，误差表/图  

### FD/ID 对比（轻量）

- （已收口）FD/ID 对比统一建议使用 `run_full_dynamics_validation`；需要调 π_fd 则用 `run_debug_pi_phys_id_fd_plot`

### 力矩 / 轨迹对比

- run_torque_comparison.m — 多版本力矩（刚体/摩擦/辨识）  
- compare_torque_sim_vs_id.m — 力矩仿真 vs ID（含对齐、摩擦、可选 FD）  
- compare_forward_dynamics_vs_measured.m — FD 积分 vs 实测  
- compare_sim_vs_real_trajectory.m — 仿真 vs 实测轨迹  
- compare_single_joint_torque_aligned.m — 单关节对齐力矩  
- run_compare_single_joint_torque.m — 单关节对比流程  
- run_cad_quick_check.m — CAD 快速检查  

### 示例

- example_run_min_param_id.m — 一键最小参数辨识（建议唯一保留的 example_run_min_param_id）  
- example_compare_torque_sim_vs_id.m  
- example_compare_single_joint_torque.m  

### 验收 / 自检

- test_min_param_URDF.m — 最小参数集自洽

## 六、建议操作小结

1. **删除或收口重复**  
   - 删除根目录 **example_run_min_param_id.m**，或改为仅调用 `test/example_run_min_param_id`。  
   - 可删 **test/run_full_dynamics_validation.asv**。

2. **不删、仅区分用途**  
   - run_light_dynamics_validation：日常主入口；run_full_dynamics_validation：最终交叉验证；run_debug_pi_phys_id_fd_plot：快速调 π_fd。  
   - run_batch_id_summary / run_batch_fd_summary：表图汇总，与全流程并存。  
   - run_torque_comparison 与 run_min_param_id_from_csv 的 3b：保留两脚本，后续如需可抽公共预处理/力矩计算。

3. **推荐顺序（与《验证代码梳理与推荐运行顺序》一致）**  
   - 阶段 A：test_ReMatrix_E1_limb_URDF → test_min_param_URDF → run_check_full_param_consistency_cad_only。  
   - 阶段 B：run_min_param_id_from_csv（或 run_iden_full_workflow）→ run_torque_comparison。  
   - 阶段 C：run_light_dynamics_validation（需要快速迭代时）/ run_full_dynamics_validation（需要交叉验证时）。  
   - 表图汇总：run_batch_id_summary → run_batch_fd_summary。

若存在 **run_fd_from_cache_plot.m**（从 fd_cache 复算并绘图），其与 run_batch_fd_summary 用途相近但更轻量（不写表、只画图），可保留作“只改 FD(π_cad) 时快速看图”的入口。

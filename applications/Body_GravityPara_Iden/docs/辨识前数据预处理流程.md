# 辨识前数据预处理流程

本文档与 `preprocess_id_data.m`、`continuous_window_id_data.m` 及 `run_min_param_id_from_csv` 中的「预处理」选项一致，用于保证 **q、q̇、q̈、τ 时间对齐、频带一致**，再构造辨识用目标力矩 **τ_id**，避免“左边很平、右边很毛”导致参数被污染。

---

## 一、为什么要先做预处理

- **时间轴**：若时间戳未对齐、首尾或中间有异常段，后面滤波再漂亮也没用。
- **运动学同源**：若 q 用原始值、q̇ 用驱动器滤波值、q̈ 再对另一量差分，三者频带不一致，辨识会变差。
- **加速度最敏感**：q̈ 若很脏，做「减转子惯量」时会把噪声一起减掉，τ_id 被破坏。
- **力矩必须滤**：电流→力矩常有高频抖动、量化噪声，需零相位低通，但不要滤得过狠。

因此推荐：**先从 q 统一得到 q/q̇/q̈（SG 求导）→ 再统一时间轴并裁剪有效段 → 对 τ 低通 → 再算 τ_id**。本仓库实现为**先 SG 求导（全段），再时间轴裁剪**。

---

## 二、最小可行流程（七步，执行顺序：先 2–3，再 1，再 4、5）

### Step 2–3（先执行）：从位置 q 得到 q_s、q̇_s、q̈_s（SG 求导）

- **推荐**：以编码器位置 q（**全段**）为唯一来源，对 q 做 **Savitzky–Golay** 求导，**一次性**得到 q_s、q̇_s、q̈_s，三者同源、频带一致。
- **方法**：SG 平滑 + 一阶/二阶导（同一多项式），例：500 Hz 时窗长 21～41 点、阶次 3 或 4。
- **不推荐**：用「原始速度再差分得加速度」作为主链路，q̈ 对噪声太敏感。

### Step 1（再执行）：统一时间轴并裁剪有效段

- **时间戳对齐**：保证同一行对应同一时刻。
- **删除首尾异常**：启动/停止瞬间明显异常段裁掉（`trim_start_s`/`trim_end_s` 或 `trim_start_n`/`trim_end_n`）。
- **去掉饱和、掉包、通信异常**：可选按 `max_abs_q`、`max_abs_qd`、`max_abs_tau` 阈值剔除；`remove_nan_inf=true` 时删掉含 NaN/Inf 的行。
- **保证 q_s、q̇_s、q̈_s、τ 同步**：裁剪后行数一致、一一对应。

### Step 4：对力矩 τ 做零相位低通

- **必须滤**：消除电流量化、驱动器波动、尖峰毛刺。
- **推荐**：**Butterworth + filtfilt**（零相位）。  
  - 例：500 Hz 时，2 阶或 4 阶，截止 10～20 Hz 试起；激励主频低（0.5～2 Hz）时 10～20 Hz 往往够用。
- **原则**：保留动力学主趋势，不保留毛刺。

### Step 5：计算辨识用目标力矩 τ_id

公式（频带一致）：

- **τ_id = τ_s − J_eq·q̈_s − τ_f(q̇_s) − τ_bias**
  - τ_s：滤波后的力矩  
  - J_eq·q̈_s：转子惯量项（用**平滑后的 q̈_s**）  
  - τ_f(q̇_s)：摩擦项（用**平滑后的 q̇_s**）  
  - τ_bias：常值偏置（若用）

这样左右两边都用同一套平滑/滤波量，避免“左边平、右边毛”。

### Step 6：用 (q_s, q̇_s, q̈_s, τ_id) 做最小参数辨识

- 将 **q_s、q̇_s、q̈_s、τ_id** 送入 `continuous_window_id_data`（并传入已算好的 qdd，不再内部重算），再交给 `identify_min` 构造 Y 并做最小二乘。

### Step 7：验证——先看 τ_Jm = J_eq·q̈

- 画出 **τ_Jm = J_eq·q̈_s**，与原始 τ 对比。
- 若 τ_Jm 高频抖得厉害、幅值在很多地方明显不合理、比原始力矩还毛，说明 **q̈ 链路还不能直接拿来做转子惯量补偿**，应先加强加速度平滑（如加大 SG 窗长、降阶等），再辨识。

## 三、本仓库中的实现

| 步骤 | 实现位置 | 说明 |
|------|----------|------|
| Step 2–3（先） | `preprocess_id_data.m` | **先 SG 求导**：`sg_order`、`sg_frame`，由全段 q 得到 q_s、q̇_s、q̈_s |
| Step 1（再） | `preprocess_id_data.m` | **再时间轴裁剪**：`t_start_s`/`t_end_s`、`trim_*`、`remove_nan_inf`、`max_abs_*`，得到裁减后 q_s、q̇_s、q̈_s、τ |
| Step 4 | `preprocess_id_data.m` | `tau_lowpass_fc_Hz`、`tau_lowpass_order`，Butterworth + filtfilt |
| Step 5 | `preprocess_id_data.m` | `J_eq`、`friction_params`、`tau_bias`，输出 τ_id 与 aux（含 τ_Jm 等） |
| Step 6 | `run_min_param_id_from_csv` | `opts.use_preprocess_id = true` 时先调 `preprocess_id_data`，再 `continuous_window_id_data(..., qdd_s)` |
| Step 7 | 用户 | 用 `preprocess_id_data` 返回的 `aux.tau_Jm` 与原始 τ 绘图对比 |

---

## 四、辨识阶段建议“至少滤”的量

1. **位置 q**：轻度平滑，用于稳定求导（本流程中由 SG 完成）。  
2. **加速度 q̈**：不单独滤，而是**通过对 q 的平滑统一得到**。  
3. **力矩 τ**：**必须低通**（本流程中 Butterworth + filtfilt）。

**速度 q̇**：最佳做法是由平滑后的 q 一起求出来（本流程即 q̇_s）；若驱动器速度质量很好，可作对比参考，但辨识主链路建议固定为“从 q 经 SG 得到”的一种。

---

## 五、为何“先减转子惯量再辨识”时更要重视 q̈ 平滑

- 补偿项里有 **J_eq·q̈**；若 q̈ 很脏，减掉的是「转子惯量项 + 加速度噪声」，会直接把 τ_id 搞坏。
- 因此**先把 q̈ 做稳**（本流程中通过「只从 q 经 SG 得 q/q̇/q̈」实现），再谈最小二乘与参数结果。

---

## 六、调用示例

```matlab
% 在 run_min_param_id_from_csv 中开启预处理
opts = struct();
opts.mode = 'continuous';
opts.continuous_opts = struct('t_start_s', 2, 't_end_s', 4);
opts.use_preprocess_id = true;
opts.prep_opts = struct(...
    'sg_order', 4, ...
    'sg_frame', 41, ...
    'tau_lowpass_fc_Hz', 15, ...
    'tau_lowpass_order', 2);
% 若需补偿：从汇总表读或手动给
% opts.load_friction_from_summary = true;
% opts.prep_opts.J_eq = opts.I_a;
% opts.prep_opts.friction_params = opts.friction_params;
run_min_param_id_from_csv('data/excitation/your_file.csv', opts);
```

单独调用预处理（不跑辨识）时，可用返回的 `aux.tau_Jm` 与原始 τ 做 Step 7 的对比图。

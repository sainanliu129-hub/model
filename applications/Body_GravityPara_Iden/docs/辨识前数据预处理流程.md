# 辨识前数据预处理流程（低通统一版）

本文档对应 `preprocess_id_data.m` / `preprocess_lowpass_id_data.m` 的当前实现。  
当前流程已统一为：**仅低通链路，不再使用 SG，不再使用多周期对齐平均**。

---

## 一、统一流程

1. 对全段 `q` 做零相位低通（Butterworth + `filtfilt`）得到 `q_s`。  
2. 由 `q_s` 中心差分得到 `qd_s`，再差分得到 `qdd_s`。  
3. 对全段 `tau` 做零相位低通（Butterworth + `filtfilt`）得到 `tau_s`。  
4. 按时间窗与异常规则裁剪有效段（对 `q_s/qd_s/qdd_s/tau_s` 同步裁剪）。  
5. 构造辨识目标：`tau_id = tau_s - J_eq*qdd_s - tau_f(qd_s) - tau_bias`。  
6. 将 `(q_s, qd_s, qdd_s, tau_id)` 送入连续窗辨识。

---

## 二、默认参数（与 `plot_preprocess_before_after_trajectories.m` 一致）

- `t_start_s = 2.1`
- `t_end_s = 4.1`
- `q_lowpass_fc_Hz = 25`
- `q_lowpass_order = 2`
- `tau_lowpass_fc_Hz = 25`
- `tau_lowpass_order = 2`

说明：
- 截止频率越低，滤波越强；越高，动态保留越多但噪声更大。
- 阶数越高滚降越陡，可能带来振铃；推荐先用 2 阶。

## 三、实现映射

- 低通预处理主函数：`principle/preprocess_lowpass_id_data.m`
- 兼容入口：`principle/preprocess_id_data.m`（内部转调低通函数）
- 解耦入口（滤波+可选补偿）：`principle/run_id_preprocess_pipeline.m`
- 连续窗：`principle/continuous_window_id_data.m`
- 数据集构建（读取+预处理+连续窗）：`principle/build_id_dataset_from_csv.m`
- 辨识分发：`principle/identify_params_dispatch.m`
- 正/逆动力学分发：`principle/forward_dynamics_dispatch.m`、`principle/inverse_dynamics_dispatch.m`

### 预处理补偿开关

- `do_compensation = false`：只做低通与导数估计，不做补偿（`tau_id≈tau_s`）。
- `do_compensation = true`：在预处理中启用 `J_eq / friction_params / tau_bias`。
- 也可设 `load_friction_from_summary = true` 自动从汇总表读取 `friction_params` 与 `I_a`（写入 `J_eq`）。

# 动力学最小参数辨识端到端方案

基于你给的「5 周期激励 → 周期对齐+平均降噪 → 最小参数辨识 → 用最小参数做正动力学」完整链路，本文档给出**与当前 E1 项目对接的可落地方案**（先方案，不实现代码）。实现时可按本文档拆函数、对变量名。

---

## 一、输入/输出定义（与项目对接）

### 1.1 输入（原始采集）

| 项目 | 说明 | 与现有代码对应 |
|------|------|----------------|
| 采样序列 | \(t_k\)，单位 s | CSV 首列 `time`，或从 `read_leg_trajectory_from_csv` 得到的 `traj.t` |
| 关节位置 | \(q_k \in \mathbb{R}^n\)，\(n=6\)（单腿）或 3（单臂） | 单腿：CSV 的 `leg_l1_joint`..`leg_l6_joint`（左腿）或 `leg_r1_joint`..`leg_r6_joint`（右腿）；列顺序与 `ReMatrix_E1_limb_URDF(limb, ...)` 一致 |
| 关节速度 | \(\dot q_k\)（可选） | 若未采集，由 \(q\) 在相位域或时间域估计（见 B4） |
| 关节力矩 | \(\tau_k \in \mathbb{R}^n\) | 需由电流/力矩换算得到，与 `ReMatrix` 的「基于电流」约定一致；若来自仿真可用 `inverseDynamics` |
| 周期数 | \(K=5\) | 与 `config.traj_cycle` / TLBO 的 `traj_cycle` 一致 |
| 周期 \(T\) | 单周期时长 (s) | 与 `config.period` / TLBO 的 `period` 一致；未知时可从数据估计 |

**数据维度约定**（与 `ReMatrix_E1_limb_URDF` 一致）：

- 单腿：`q`, `qd`, `tau` 为 **N×6**（每行一个时刻，列对应 6 关节）。
- 回归矩阵调用：`Y = ReMatrix_E1_limb_URDF(limb, qfull, qdfull, qddfull, output_type, para_order)`，其中 `qfull/qdfull/qddfull` 为 **N×6**；`Y` 为 **(N×6)×60**（output_type=1），即每时刻 6 行、共 N×6 行，列数为 60 全参。

### 1.2 输出（最终要用的）

1. **周期平均后的数据集**  
   \(\bar q(\phi),\ \overline{\dot q}(\phi),\ \overline{\ddot q}(\phi),\ \bar\tau(\phi)\)，\(\phi\in[0,1)\) 的离散网格 \(\phi_j=j/M\)，\(j=0,\ldots,M-1\)。  
   可选：\(\sigma_\tau(\phi_j)\) 用于 WLS 权重。

2. **最小参数辨识结果**  
   \(\hat X = \theta_{\min}\)（\(p_{\min}\) 维），满足 \(\tau \approx Y_{\min}(q,\dot q,\ddot q)\,\hat X\)。  
   与现有 `rref` 得到的 `index_base` 对应：\(Y_{\min}=Y(:, \texttt{index\_base})\)。

3. **正动力学函数**  
   给定 \((q,\dot q,\tau)\) 输出 \(\ddot q\)，内部用 \(Y_{\min}\) 与 \(\hat X\) 构造 \(M(q)\)、\(h(q,\dot q)\) 后解 \(\ddot q = M^{-1}(\tau - h)\)，**不依赖 URDF 的完整惯性参数**。

## 二、周期对齐 + 平均降噪（相位域，非时间对齐）

核心：把每个周期映射到**同一相位轴** \(\phi\in[0,1)\)，再按相位平均，降噪且相位一致。

### 2.1 Step B1：周期切分（把整段切成 K=5 个周期）

用**相位锚点**切周期（比硬按 \(T\) 更鲁棒）。

- **锚点方式（任选一种，选最稳的信号）**  
  - **零交叉上升沿**：选一个关节 \(q_r\)（激励幅值大、噪声小），找 \(q_r\) 从负到正穿越 0 且 \(\dot q_r>0\) 的时刻。  
  - **峰值点**：找 \(q_r\) 的局部最大值，相邻间隔约 \(T\)。  
  - **若记录了轨迹生成器的 phase**：可直接用 phase=0 作为周期起点。  

得到周期起点索引：\(k_0,k_1,\ldots,k_K\)（共 \(K+1\) 个点），第 \(c\) 周期为 \([k_{c-1},\,k_c)\)。

- **锚点去噪与约束（避免噪声多次触发）**  
  - 对锚点信号 \(q_r\) 先做**轻滤波**（或对 \(\dot q_r\) 用 SG），再检测锚点。  
  - **最小间隔约束**：相邻锚点时间间隔必须在 \([0.8T,\,1.2T]\) 内（或按采样点数阈值等价约束），否则丢弃或合并，保证 \(K=5\) 切得稳。  
  - 若用**零交叉**：要求 \(|q_r|\) 在交叉点附近单调，且 \(\dot q_r\) 大于一给定阈值，再记为有效上升沿。
- **建议**：第 1 周期常有热机/控制器收敛效应，先保留；完成平均后可视情况剔除异常周期（见 B3）。

### 2.2 Step B2：每周期映射到统一相位网格（相位重采样）

各周期样本数可能不同，不能按样本号直接平均。

1. **统一相位网格**  
   \(\phi_j = j/M,\ j=0,\ldots,M-1\)。建议 \(M \approx\) 每周期平均样本数，或固定如 1000。  
   **可选**：若提供生成激励时的单周期时间与位置（ref_t, ref_q），则用 \(\phi = \texttt{ref\_t}/T\) 作为相位网格，与设计轨迹完全一致，重采样到该网格后平均。

2. **第 \(c\) 周期相位**  
   \(\phi^{(c)}(k) = (k - k_{c-1})/(k_c - k_{c-1})\)，\(k\in[k_{c-1}, k_c)\)。

3. **插值到 \(\phi_j\)**  
   对 \(q^{(c)},\ \dot q^{(c)},\ \tau^{(c)}\) 在 \(\phi^{(c)}(k)\) 上已知，插值到 \(\phi_j\)。  
   建议 **pchip**（保形、减少过冲）；线性也可。  
   得到每周期在统一网格上的：\(q^{(c)}(\phi_j),\ \dot q^{(c)}(\phi_j),\ \tau^{(c)}(\phi_j)\)。

4. **端点与闭环一致**  
   - 相位网格使用 **[0, 1)** 采样（\(\phi_j=j/M,\ j=0,\ldots,M-1\)），**不**把 \(\phi=1\) 单独再取一点，避免与 \(\phi=0\) 重复。  
   - 若周期切分为 \([k_{c-1}, k_c)\)，则每周期**最后一帧**（\(k_c\) 对应的样本）不参与该周期内的相位映射，这样 \(\phi=1\) 不会重复计入，闭环一致。

### 2.3 Step B3：周期平均 + 方差

对 \(c=1..K\)：

- **均值**  
  \(\bar q(\phi_j) = \frac{1}{K}\sum_{c=1}^K q^{(c)}(\phi_j)\)，同理 \(\overline{\dot q}(\phi_j),\ \bar\tau(\phi_j)\)。

- **方差（用于 WLS 权重）**  
  \(\sigma_\tau^2(\phi_j) = \frac{1}{K-1}\sum_{c=1}^K\big(\tau^{(c)}(\phi_j)-\bar\tau(\phi_j)\big)^2\)，或每关节单独 \(\sigma_{\tau,i}^2(\phi_j)\)。

若某周期整体明显偏离（如整段力矩偏大/偏小），可视为异常周期剔除后再平均。

### 2.4 Step B4：计算 \(\overline{\ddot q}\)（强烈建议在均值上求）

两种常见做法（当前实现采用推荐做法）：

- **做法 A（仅平均关节角）**：只对关节角按相位平均得到 \(\bar q(\phi)\)；速度、加速度由 \(\bar q\) 在相位域差分（或 SG 求导）后乘 \(1/T\)、\(1/T^2\) 得到。适用于**未采集 qd** 时。
- **做法 B（平均关节角 + 平均速度，推荐有 qd 时）**：\(\bar q = \frac{1}{K}\sum q^{(c)}\)，\(\overline{\dot q} = \frac{1}{K}\sum \dot q^{(c)}\)；加速度由**对平均速度** \(\overline{\dot q}\) 在相位域求导后乘 \(1/T\) 得到，即 \(\overline{\ddot q} = \frac{1}{T}\frac{\mathrm{d}\overline{\dot q}}{\mathrm{d}\phi}\)。采集了 qd 时优先用做法 B，避免对 q 求二阶导放大噪声。

- **若有 \(\overline{\dot q}(\phi)\)**：在相位域做平滑差分或 SG 滤波得到 \(\frac{\mathrm{d}}{\mathrm{d}\phi}\overline{\dot q}\)，再乘 **时间尺度** 得到 \(\overline{\ddot q}\)。  
- **若没有 \(\dot q\)**：可对 \(\bar q(\phi)\) 做 SG 滤波同时得到 \(\overline{\dot q},\ \overline{\ddot q}\)（仍在相位域求导后乘尺度）。
- **数值方法建议**：  
  - **推荐**：SG 滤波**直接在 \(\phi\) 轴上**求一阶/二阶导，比简单差分更稳。  
  - 若用差分：用**中心差分**；对**端点**用一阶差分或**镜像延拓**再差分的方式，避免边界放大噪声。

**关键：相位→时间的导数尺度**  
设 \(\phi = t/T\)，则  
\(\frac{\mathrm{d}}{\mathrm{d}t} = \frac{1}{T}\frac{\mathrm{d}}{\mathrm{d}\phi}\)，  
\(\ddot q = \frac{1}{T^2}\frac{\mathrm{d}^2 q}{\mathrm{d}\phi^2}\)。  
在相位轴差分时，必须乘上 \(1/T\)（速度）和 \(1/T^2\)（加速度）。

## 三、最小参数辨识（不考虑摩擦）

现有形式：\(\tau = Y(q,\dot q,\ddot q)\,\theta\)，\(Y\) 为 **(N·n)×60**；经 **列归一化 + rref** 得到线性无关列索引 `index_base`，\(Y_{\min}=Y(:, \texttt{index\_base})\)，\(\tau = Y_{\min}\,X\)。

### 3.1 Step C1：组装辨识矩阵

- 对平均后的每个相位点 \(\phi_j\)：  
  \(A_j = Y_{\min}(\bar q_j,\, \overline{\dot q}_j,\, \overline{\ddot q}_j)\)。  
  其中 \(Y_{\min}\) 由**同一套** `index_base` 得到（见下）。
- **\(Y_{\min}\) 与 index_base 的固定方式（保证可复现）**：  
  - **列归一化规则固定**：例如对每列除以其 **2 范数**（或列最大绝对值），并**保存该缩放**用于后续一致性；至少保证每次做 rref 的输入矩阵一致。  
  - **index_base 只算一次**：在**一条代表性数据集**上算一次（建议就是用平均后的轨迹 \(\bar q,\overline{\dot q},\overline{\ddot q}\)），得到 **index_base** 后，后续辨识与 FD **全部复用**这一套；同一 limb、同一 para_order 即可复现。  
  - 流程：先算 \(Y = \texttt{ReMatrix\_E1\_limb\_URDF}(\texttt{limb}, \bar q, \overline{\dot q}, \overline{\ddot q}, 1, \texttt{para\_order})\)，再列归一化（固定规则）→ **rref** → **index_base**；之后所有点都用 \(Y(:, \texttt{index\_base})\) 作为 \(Y_{\min}\)。
- **堆叠与 \(b\) 的行顺序（与 ReMatrix 严格一致）**：  
  ReMatrix 输出是按**时刻**堆叠：第 1 帧 n 行、第 2 帧 n 行、…、第 M 帧 n 行。因此 \(b\) 必须按**同样顺序**堆成列向量：
  \[
  b = [\bar\tau(1,:)^\top;\ \bar\tau(2,:)^\top;\ \ldots;\ \bar\tau(M,:)^\top],
  \]
  即 `b = tau_bar(:)`（按行拉成列），与 \(A\) 的每一行一一对应。否则最小二乘“看起来能解但结果错”。

与现有代码对应：  
- 单点调用：`Y_one = ReMatrix_E1_limb_URDF(limb, q, qd, qdd, 1, para_order)` 得到 6×60；多时刻堆叠后 (M·6)×60，再取 `Y_min = Y(:, index_base)` 得到 (M·6)×p_min。  
- \(b\)：`b = tau_bar(:)`，与 \(A\) 行一一对应（第 1 帧 6 行对应 \(\tau(1,:)\) 的 6 个分量，依此类推）。

### 3.2 Step C2：求解（推荐加权最小二乘）

- 目标：\(\hat X = \arg\min_X \|W(AX-b)\|_2^2\)。  
- 权重 \(W = \mathrm{diag}(w)\)：  
  \(w(\phi_j)\) 可取 \(1/(\sigma_\tau(\phi_j)+\epsilon)\)（若按关节加权则每行对应一个关节的 \(\sigma_{\tau,i}\)）。  
- 若 \(A\) 条件数很大，可加正则：\(\min \|W(AX-b)\|^2 + \lambda\|X-X_0\|^2\)，\(X_0\) 可由 URDF 的 \(\theta\) 经同一 `index_base` 取子向量得到（`get_limb_theta_from_URDF` 得到 60×1，再 `theta(index_base)`）；没有先验时可先不加。

### 3.3 Step C3：验证（必须做）

- 预测力矩：\(\tau_{\mathrm{pred},j} = Y_{\min}(\bar q_j,\overline{\dot q}_j,\overline{\ddot q}_j)\,\hat X\)。  
- 检查每关节 RMSE、最大误差；误差是否集中在换向相位（换向处噪声通常更大）。

## 四、用最小参数 \(\hat X\) 做正动力学（FD）

结论：最小参数集不能唯一还原每个 link 的 \(m,c,I\)，但可以用 **ID 构造 \(M(q)\) 和 \(h(q,\dot q)\)**，仅依赖 \(Y_{\min}\) 与 \(\hat X\) 做 FD。

定义**最小参数逆动力学**：  
\(\mathrm{ID}_{\min}(q,\dot q,\ddot q) = Y_{\min}(q,\dot q,\ddot q)\,\hat X\)。  
其中 \(Y_{\min}\) 对单点 \((q,\dot q,\ddot q)\) 的调用：先算 \(Y_{\mathrm{one}} = \texttt{ReMatrix\_E1\_limb\_URDF}(\texttt{limb}, q, qd, qdd, 1, \texttt{para\_order})\)（这里 \(q,qd,qdd\) 为 1×n），再取 \(Y_{\mathrm{one}}(:, \texttt{index\_base})\) 得到 1×p_min，乘 \(\hat X\) 得 n×1。

### 4.1 Step D1：偏置项 \(h(q,\dot q)\)

\[
h(q,\dot q) = \mathrm{ID}_{\min}(q,\dot q,\,0).
\]

### 4.2 Step D2：质量矩阵 \(M(q)\)（列构造）

- \(g(q) = \mathrm{ID}_{\min}(q,0,0)\)。  
- 对 \(i=1..n\)：\(M(:,i) = \mathrm{ID}_{\min}(q,0,e_i) - g(q)\)，其中 \(e_i\) 为第 \(i\) 个标准基向量。  
- **注意**：构造 \(M(q)\) 时**必须** \(\dot q=0\)，否则会混入科氏项。
- **数值稳健性（工程建议）**：  
  - **对称化**：理论上 \(M\) 对称，数值上可做 \(M \leftarrow (M+M^\top)/2\)。  
  - **正定性检查**：若出现极小负特征值（数值误差），可做轻微抬升 \(M \leftarrow M + \delta I\)（\(\delta\) 很小，仅作数值稳定），避免求解时发散。

### 4.3 Step D3：求解 \(\ddot q\)

\[
\ddot q = M(q)\setminus\big(\tau - h(q,\dot q)\big).
\]  
工程上用反斜杠线性求解，不显式求逆。

与现有 `compute_e1_limb_forward_dynamics(limb, q, qd, tau)` 的关系：  
- 现有函数依赖 URDF 的完整模型；本方案提供**仅依赖辨识结果 \(\hat X\) 与 \(Y_{\min}\)** 的 FD，适用于「只有最小参数、无完整惯性参数」的场景。

## 五、建议的工程实现结构（拆函数串起来）

### 1）`cycle_average(raw) -> avg_data`

- **输入**：原始日志（\(t\), \(q\), \(\dot q\)（可选）, \(\tau\)），以及周期数 \(K\)、周期 \(T\)（或由锚点估计）、锚点选项（零交叉/峰值/相位）。  
- **输出**：  
  - `phi`：统一相位网格 (M×1)；  
  - `q_bar, qd_bar, qdd_bar, tau_bar`：M×n；  
  - `tau_std`：M×n 或 M×1，用于权重；  
  - 可选：每周期重采样后的 `q_cyc{c}` 等，便于检查。

### 2）`identify_min(avg_data, limb, opts) -> X_hat, index_base, metrics`

- **输入**：`avg_data`（含 `phi, q_bar, qd_bar, qdd_bar, tau_bar, tau_std`）、`limb`、可选 `opts`（如 `para_order`, 是否 WLS、是否正则、\(X_0\)）。  
- **内部**：用 `ReMatrix_E1_limb_URDF` 在平均轨迹上算全矩阵 \(Y\)，列归一化 + rref 得到 **index_base**；组装 \(A,b\)，WLS 求解 \(\hat X\)。  
- **输出**：`X_hat`（p_min×1）、`index_base`（用于后续 FD）、拟合指标（RMSE、每关节误差等）。

### 3）`forward_dynamics_min(q, qd, tau, X_hat, index_base, limb) -> qdd`

- **内部**：  
  - \(\mathrm{ID}_{\min} = (q,qd,qdd) \mapsto Y_{\min}(q,qd,qdd)\,\hat X\)（单点调用 ReMatrix 再取列 index_base）；  
  - \(h(q,qd) = \mathrm{ID}_{\min}(q,qd,0)\)；  
  - \(g(q) = \mathrm{ID}_{\min}(q,0,0)\)；  
  - \(M(:,i) = \mathrm{ID}_{\min}(q,0,e_i) - g(q)\)；  
  - \(\mathtt{qdd} = M\setminus(\tau - h)\)。  
- **输出**：`qdd` (n×1)。

## 六、最容易翻车点与实现细节（项目内强调）

1. **相位导数尺度**  
   在相位轴差分求 \(\dot q,\ddot q\) 时，必须乘 \(1/T\) 和 \(1/T^2\)。

2. **构造 \(M(q)\) 时 \(\dot q=0\)**  
   否则会混入科氏/向心项，\(M\) 不再表示惯性矩阵。

3. **平均顺序**  
   必须先**按锚点切周期 → 每周期相位重采样 → 再对相位点平均**；不要对原始时间序列直接平均。

4. **第 1 周期与异常周期**  
   建议先画 5 周期叠图，必要时剔除第 1 周期或明显偏离的周期后再平均。

5. **实现时的 6 处细节**  
   锚点去噪与最小间隔 \([0.8T,1.2T]\)、相位端点与闭环不重复 \(\phi=1\)、相位域求导用 SG/中心差分+端点处理、**index_base 固定归一化且只算一次**、**\(b\) 与 ReMatrix 行顺序一致**、FD 的 \(M\) 对称化与正定性检查（见上文对应小节）。

6. **用生成激励作相位网格**  
   生成激励时（如 `run_excitation_trajectory_standalone`）会保存单周期 `ref_t, ref_q, ref_qd` 到与 CSV 同目录的 `[基名]_ref.mat`。辨识时把该 CSV 与同目录的 `*_ref.mat` 一起放到 `data/excitation/`，`run_min_param_id_from_csv` 会自动加载；或通过 `opts.ref_t, opts.ref_q, opts.ref_qd` 传入。`cycle_average` 用 \(\phi=\texttt{ref\_t}/T\) 作为相位网格，与设计轨迹一致。

## 八、小结

按顺序实现以下三步即可串通整条链路：

1. **周期对齐 + 平均**：锚点切周期 → 相位重采样（pchip）→ 相位平均 → 在均值上求 \(\overline{\ddot q}\)（注意 \(1/T^2\) 尺度）。  
2. **最小参数辨识**：固定 `index_base`（由平均轨迹 rref 得到）→ 堆叠 \(A,b\) → WLS（可选正则）→ \(\hat X\)，并做 \(\tau_{\mathrm{pred}}\) 验证。  
3. **FD**：用 \(\mathrm{ID}_{\min}\) 构造 \(h\) 和 \(M\)（\(M\) 用 \(\dot q=0\)），解 \(\ddot q = M\setminus(\tau - h)\)。

若你下一步需要「按你们项目变量名写完整 MATLAB 骨架」（含周期锚点检测、相位重采样、WLS、FD），只需确认两件事：  
- 日志字段名与维度（如 `t, q, qd, tau` 的矩阵形状及单腿列索引）；  
- `Ymin_fun` 的调用方式（本项目即 `ReMatrix_E1_limb_URDF` 取列 `index_base`，单点输入 1×n，输出 n×1 的 \(\tau\)）。

## 九、采集数据存放与调用

### 9.1 数据文件放在哪里

- **推荐目录**（与现有 Body_GravityPara_Iden 脚本一致）：  
  `applications/Body_GravityPara_Iden/data/` 下按用途分子目录，例如：
  - **激励辨识用**：`data/excitation/`（5 周期激励轨迹采集的 CSV）
  - 其他：`data/跑步/`、`data/dynamics/` 等（与 `example_compare_torque_sim_vs_id.m` 等一致）

- **文件格式**：与 `read_leg_joint_csv` 兼容的 CSV（见 `utility_function/read_leg_joint_csv.m`）  
  - 老格式：表头含 `time, pos_leg_l1_joint, vel_leg_l1_joint, torque_leg_l1_joint, ...`（左腿 l1..l6、右腿 r1..r6）  
  - 新格式：`time, q_1_data, ..., qd_1_data, ..., tau_1_data, ...`  
  辨识单腿时只用到**左腿 6 列**：`pos_leg_l`（即 q）、`vel_leg_l`（qd，可选）、`torque_leg_l`（τ）。

- **采集要求**：同一段数据内包含 **K=5 个周期** 的激励，周期时长 T 与生成轨迹时的 `config.period` 一致（如 8 s）；CSV 中需有 **时间 + 关节位置 + 关节力矩**（若仅有位置，可用仿真 ID 生成 τ 做流程验证）。

### 9.2 怎么调用函数

1. **保证路径**：在运行脚本前执行 `ensure_body_gravity_para_iden_path()`（或把 `applications/Body_GravityPara_Iden`、`robot_model`、`dynamics`、`utility_function` 加入路径），以便使用 `read_leg_joint_csv`、`ReMatrix_E1_limb_URDF`、`cycle_average`、`identify_min`、`forward_dynamics_min`。

2. **读入数据**：用 `read_leg_joint_csv(csv_file)` 得到 `data`，取单腿：
   - `t = data.time;`（N×1）
   - `q = data.pos_leg_l;`（N×6）
   - `qd = data.vel_leg_l;`（N×6，可选）
   - `tau = data.torque_leg_l;`（N×6）

3. **三步调用**（与第五节对应）：
   - **周期平均**：`avg_data = cycle_average(t, q, tau, K, T, 'qd', qd);`（无 qd 可省略 'qd', qd）
   - **最小参数辨识**：`[X_hat, index_base, metrics] = identify_min(avg_data, 'left_leg');`
   - **正动力学**（任选一点验证）：`qdd = forward_dynamics_min(q(k,:)', qd(k,:)', tau(k,:)', X_hat, index_base, 'left_leg', 1);`

4. **示例脚本**：`run_min_param_id_from_csv.m` 已提供完整示例：指定 CSV 路径（默认从 `data/excitation/` 或 `data/跑步/` 读）、执行上述三步、画辨识误差图、可选保存 `X_hat`/`index_base` 到 `.mat` 供 FD 复用。

## 十、连续窗辨识（单周期 + 衔接段）

工程上常用「一个稳定周期 + 起始/结束衔接段」做辨识，**不需要识别周期**，只要把衔接段当成可能污染的非稳态做筛选/降权即可。

### 10.1 数据分段与建议

- **稳定周期段**：全部保留。
- **起始/结束衔接段**：只保留中间平滑部分，掐掉最容易出问题的点：
  - 刚启动的前 0.2～0.5 s（或前 5～10% 点）；
  - 刚结束的后 0.2～0.5 s（或后 5～10% 点）；
  - 任何明显饱和/限幅/速度突变点。

本质是避免把「控制器收敛/切换过程」当动力学本体去拟合。

### 10.2 衔接段对辨识的影响（无摩擦时）

- \(\ddot q\) 里易夹杂大尖峰（切换/跟踪拉扯）；
- \(\tau\) 里有控制器额外动作。

**建议两条硬规则**：  
① 用 SG/低通让 \(\ddot q\) 无尖峰（尤其衔接段）；  
② 对 \(|\ddot q|\) 最大的那一小部分点（如 top 1～3%）直接降权或剔除。

### 10.3 实现：`continuous_window_id_data` + `opts.mode = 'continuous'`

- **调用方式**：  
  `run_min_param_id_from_csv(csv_file, struct('mode', 'continuous', 'continuous_opts', struct(...)));`

- **continuous_opts**（可选）：
  - `trim_start_s`、`trim_end_s`：起始/结束裁掉时长 (s)，默认 0.2；
  - `trim_ratio`：若 >0，则按总时长的比例裁（覆盖上面两档）；
  - `qdd_smooth_half`：SG 滤波半窗长（点数），0=不平滑，默认 5；
  - `downweight_qdd_top_ratio`：对 \(|\ddot q|\) 最大的该比例样本在 WLS 中降权，默认 0.02（top 2%）。

- **流程**：整段数据 → 掐头去尾得时间窗 → 由 \(\dot q\) 中心差分得 \(\ddot q\) → 可选 SG 平滑 → 对高 \(|\ddot q|\) 点放大 `tau_std`（WLS 权小）→ 堆叠 \(Y_{\min}\) 与 \(\tau\)，同一套 `identify_min` 解 \(\hat X\)。FD 仍用 \(\mathrm{ID}_{\min}\) 构造 \(M,h\)，与是否做周期平均无关。

### 10.4 更稳的用法：仅用单周期辨识，衔接段只做验证

若希望参数更干净（尤其 PD 小、衔接误差大时）：

- **辨识**：只用你选定的稳定周期（例如 `cycle_average` 下 K=1，或连续窗里把 `trim_start_s`/`trim_end_s` 设大，只留中间一段）；
- **验证**：用起始/结束衔接段算预测误差，看模型在非稳态下偏差多大。

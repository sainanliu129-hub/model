# 测试代码功能总览

本文档用于快速说明 `applications/Body_GravityPara_Iden/test` 下各测试脚本的作用、推荐场景与当前关系（解耦版）。

---

## 一、推荐主入口（优先使用）

- `run_full_dynamics_validation.m`  
  - **作用**：解耦版全流程验证（数据集构建 -> 最小参数辨识 -> full 参数恢复/辨识 -> ID/FD 对比）。  
  - **适用**：你要完整评估一条数据在 ID/FD 上的效果时。  
  - **输出**：控制台 RMSE、对比图、`min_param_id_result.mat`、验证汇总 Excel。

- `run_min_param_id_from_csv.m`  
  - **作用**：从 CSV 跑“低通预处理 + 连续窗 + 最小参数辨识”。  
  - **适用**：只想重跑最小参数，不做 full 参数优化时。  
  - **输出**：`X_hat`、`index_base`、`metrics`、`prep_opts`。

- `run_torque_comparison.m`  
  - **作用**：力矩多版本对比（`tau_meas` / `NE刚体` / `NE+补偿` / `id_pred`）。  
  - **适用**：快速看力矩链路误差来源。  
  - **输出**：对比图、误差图、`torque_comparison_summary.xlsx`。

- `run_debug_pi_phys_id_fd_plot.m`  
  - **作用**：单独调 `pi_fd`（FD 导向 full 参数）并画 ID/FD 对比。  
  - **适用**：你在调 `identify_full_params_for_fd` 参数时。

- `run_decoupled_id_fd_pipeline.m`  
  - **作用**：最小示例脚本，演示“解耦接口如何串起来”。  
  - **适用**：开发新流程、写新实验脚本时作为模板。

---

## 二、数据处理与辨识辅助类

- `run_iden_full_workflow.m`  
  - **作用**：一体化“薄封装”入口（主要是参数 preset + 调 `run_min_param_id_from_csv`）。  
  - **适用**：固定配置快速复现。

- `run_light_dynamics_validation.m`  
  - **作用**：轻量批处理入口，可切换是否重辨识、是否换评估轨迹。  
  - **适用**：追求运行效率、批量比较模型配置时。  
  - **输出**：`light_validation_result.mat`、`light_validation_summary.xlsx`。

- `run_identify_pi_fd_only.m`  
  - **作用**：只做参数辨识（尤其 `pi_fd`），不做完整对比流程。  
  - **适用**：单独调 full 参数优化。

- `run_plot_id_fd_only.m`  
  - **作用**：只画 ID/FD 对比（基于已有参数集）。  
  - **适用**：参数已定，只做可视化与报告。

---

## 三、一致性/诊断/验收类

- `test_ReMatrix_E1_limb_URDF.m`  
  - **作用**：回归矩阵与动力学链路综合验收（全参、最小参数、FD 对比）。  
  - **适用**：改了 `ReMatrix` 或参数映射后做回归检查。

- `run_check_full_param_consistency.m`  
  - **作用**：基于现有轨迹检查全参数链路一致性。  

- `run_check_full_param_consistency_cad_only.m`  
  - **作用**：不依赖辨识结果，仅用 CAD 参数做链路自检。  

- `run_cad_quick_check.m`  
  - **作用**：CAD-only 快速检查（参数顺序、列范数可观性）。  

- `run_diagnose_pi_Y_columns.m`  
  - **作用**：排查 `pi` 与 `Y` 列顺序/可观性问题。  

- `verify_error_source.m`、`run_check_regressor_column_norms.m`、`run_check_pi_order_vs_urdf.m`  
  - **作用**：专项误差来源与列级诊断。

---

## 四、对比分析类脚本

- `compare_torque_sim_vs_id.m`、`compare_single_joint_torque_aligned.m`、`run_compare_single_joint_torque.m`  
  - **作用**：力矩对齐与单/多关节对比分析。  

- `compare_forward_dynamics_vs_measured.m`  
  - **作用**：正动力学积分轨迹 vs 实测轨迹对比。  

- `compare_sim_vs_real_trajectory.m`、`plot_compare_two_csv_6dof.m`  
  - **作用**：两组轨迹（仿真/实测或双CSV）6关节对比。

---

## 五、批量与导出类

- `run_batch_id_summary.m`、`run_batch_fd_summary.m`  
  - **作用**：批量汇总 ID/FD 指标。  

- `export_torque_from_leg_csv.m`  
  - **作用**：从 CSV 导出/整理力矩数据。

---

## 六、轨迹生成与示例类

- `generate_e1_test_trajectories.m`、`run_generate_e1_test_trajectories.m`  
  - **作用**：生成测试轨迹。  

- `example_run_min_param_id.m`、`example_compare_torque_sim_vs_id.m`、`example_compare_single_joint_torque.m`  
  - **作用**：示例调用脚本（教学/快速上手）。

---

## 七、当前建议使用顺序

1. 日常辨识：`run_min_param_id_from_csv.m`  
2. 全流程评估：`run_full_dynamics_validation.m`  
3. 调 full 参数优化：`run_debug_pi_phys_id_fd_plot.m`  
4. 做批量统计：`run_light_dynamics_validation.m` / `run_batch_*`

---

## 八、已收敛的公共函数（避免重复代码）

- 数据处理：`build_id_dataset_from_csv.m`、`run_id_preprocess_pipeline.m`
- 辨识分发：`identify_params_dispatch.m`
- 动力学分发：`forward_dynamics_dispatch.m`、`inverse_dynamics_dispatch.m`
- 绘图复用：`plot_compare_with_error_6dof.m`
- 导表复用：`export_validation_summary_excel.m`、`rmse_struct_to_table.m`

建议后续新增测试脚本时，优先复用以上函数，不再在脚本内重复实现。

---

## 九、脚本依赖关系图

```text
run_iden_full_workflow
  -> run_min_param_id_from_csv
      -> run_id_preprocess_pipeline
          -> preprocess_id_data
              -> preprocess_lowpass_id_data
      -> continuous_window_id_data
      -> identify_params_dispatch(method='min_ls')
          -> identify_min
      -> inverse_dynamics_dispatch / forward_dynamics_dispatch (局部验证)

run_full_dynamics_validation
  -> build_id_dataset_from_csv
      -> run_id_preprocess_pipeline -> preprocess_id_data -> preprocess_lowpass_id_data
      -> continuous_window_id_data
  -> identify_params_dispatch(method='min_ls' / 'full_fd')
      -> identify_min / identify_full_params_for_fd
  -> recover_full_params_from_beta / solve_full_params_physical
  -> inverse_dynamics_dispatch / forward_dynamics_dispatch
  -> plot_compare_with_error_6dof
  -> export_validation_summary_excel

run_torque_comparison
  -> build_id_dataset_from_csv
  -> identify_params_dispatch(method='min_ls')
  -> inverse_dynamics_dispatch
  -> plot_compare_with_error_6dof
  -> export_validation_summary_excel

run_debug_pi_phys_id_fd_plot
  -> identify_full_params_for_fd
  -> inverse_dynamics_dispatch / forward_dynamics_dispatch
  -> plot_compare_with_error_6dof

run_plot_id_fd_only
  -> load(id_param_sets.mat)
  -> inverse_dynamics_dispatch / forward_dynamics_dispatch
  -> plot_compare_with_error_6dof
  -> export_validation_summary_excel (可选)

run_batch_id_summary / run_batch_fd_summary
  -> run_plot_id_fd_only(do_plot=false)
  -> export_validation_summary_excel
```
